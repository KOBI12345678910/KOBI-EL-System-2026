```

Check EACH flow end-to-end and fix any broken link.

=== FLOW 1: QUOTE → WORK ORDER → INVOICE → PAYMENT ===

Step 1: Quote created
□ Quote saved to DB with status DRAFT

□ Customer's quote count updates on customer profile
□ Quote total calculates correctly in agorot

  - Customer auto-filled from quote
  - Product type auto-filled
  - Description auto-filled
  - Amount auto-filled

Step 4: Work order created from quote
□ work_order.source_quote_id = quote.id
□ Quote page shows link to work order: "הזמנת עבודה: WO-001"
□ Work order page shows link back to quote: "מבוסס על: QT-001"

Step 5: Work order delivered
□ Status = DELIVERED
□ "צור חשבונית" button appears
□ Clicking opens invoice form PRE-FILLED:
  - Customer from work order

  - Work order linked

□ Customer open_balance_agorot decreases
□ Payment appears in /payments list

=== FLOW 2: EMPLOYEE → SHIFT → ATTENDANCE → PAYROLL ===

Step 1: Employee created
□ Employee appears in /employees list
□ Employee selectable in shift assignment dropdown
□ Employee selectable in work order assignment
□ Employee gets login credentials

Step 2: Shift assigned
□ Shift appears in /shifts weekly view
□ Employee can see their shift in MyShiftsPage
□ SMS/email sent to employee (if enabled)

□ Clock-in creates attendance record

□ All attendance records for month aggregated

□ Payslip generated with correct amounts in agorot
□ Payroll status = PAID after approval

□ Reserved materials deducted from actual inventory

Step 4: If stock falls below minimum
□ Alert created in notification table
□ Procurement manager notified (in-app + email)
□ Auto-draft purchase order created (if configured)

Step 5: Purchase order received
□ inventory_movements record created (type: IN)
□ Inventory quantity increases
□ Moving average cost recalculated
□ Low stock alert resolved

=== FLOW 4: CUSTOMER → FINANCIAL SUMMARY ===

□ Total revenue = sum of all PAID invoices
□ Open balance = sum of (invoice totals - payments received)

□ Work order count = number of work orders
□ Last activity date = most recent of: quote/invoice/payment/work-order

□ Revenue bar chart = actual monthly totals from DB
□ Work order status donut = actual counts from DB

VERIFY: Change a value in DB → dashboard refreshes to show updated value (within 5 min auto-refresh or manual refresh).

FIX all broken data flows found.
```
