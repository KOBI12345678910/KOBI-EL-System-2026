עכשיו אנחנו בונים רק את שלב 1 במחלקת רכש:
מודול ספקים בסיסי.

חשוב מאוד:
אל תבנה שום שלב נוסף.
אל תעבור לבקשות רכש, הזמנות רכש, מלאי או מחירים.
תבנה רק את מודול הספקים הבסיסי, בצורה אמיתית, מסודרת, תקינה, עם בדיקות ואישור.

המטרה:
לבנות מודול ספקים שמתאים במיוחד למפעל מסגרות, ברזל, אלומיניום, זכוכית ונירוסטה.

המודול חייב להיות:
- בעברית מלאה RTL
- מחובר למסד נתונים אמיתי
- עם CRUD מלא
- עם חיפוש
- עם סינון
- עם כרטיס ספק
- עם נתוני דמו אמיתיים מעולם המפעל

==================================
1. מה צריך לבנות בשלב הזה
==================================

לבנות רק את הרכיבים הבאים:

1. טבלת suppliers במסד הנתונים
2. מסך רשימת ספקים
3. כפתור "הוסף ספק"
4. טופס הוספת ספק
5. טופס עריכת ספק
6. עמוד / חלונית "כרטיס ספק"
7. חיפוש בסיסי
8. סינון בסיסי
9. נתוני דמו אמיתיים
10. חיבור מלא בין UI ל-DB

==================================
2. מבנה הספקים שהמערכת צריכה לתמוך בהם
==================================

המערכת צריכה להיות מותאמת לספקים מסוגים כמו:
- ספקי ברזל
- ספקי אלומיניום
- ספקי זכוכית
- ספקי נירוסטה
- ספקי פרזול
- ספקי צבע / צביעה
- ספקי אביזרים
- ספקי מסילות / מנועים / גלגלים
- ספקי ברגים / עוגנים / מחברים
- ספקי עבודות חוץ / קבלני משנה
- ספקי הובלה ולוגיסטיקה

==================================
3. טבלת database שחייבת להיבנות
==================================

צור טבלת suppliers עם השדות הבאים:

- id
- supplier_number
- supplier_name
- contact_person
- phone
- mobile
- email
- address
- city
- category
- supply_type
- payment_terms
- lead_time_days
- vat_number
- status
- notes
- created_at
- updated_at

הנחיות:
- id = primary key
- supplier_number = מספר ספק ייחודי
- supplier_name = חובה
- status = ברירת מחדל "פעיל"
- created_at / updated_at = אוטומטיים

6. מסך רשימת ספקים
==================================

בנה מסך "ספקים" עם:

כותרת:
"ניהול ספקים"

רכיבים במסך:
- כותרת עמוד
- כפתור "הוסף ספק"
- שורת חיפוש
- פילטר לפי סטטוס
- פילטר לפי קטגוריית ספק
- טבלת ספקים

עמודות בטבלה:
- מספר ספק
- שם ספק
- איש קשר
- טלפון
- עיר
- קטגוריית ספק
- זמן אספקה
- סטטוס
- פעולות

בפעולות:
- צפייה
- עריכה
- מחיקה

דרישות טבלה:
- RTL מלא
- מיון בסיסי
- חיפוש לפי שם ספק / איש קשר / טלפון
- סינון לפי סטטוס
- סינון לפי קטגוריה
- תצוגה נקייה ומקצועית

==================================
13. בדיקות שחובה לבצע לפני שאתה מסיים
==================================

לפני שאתה אומר שסיימת, חובה לבדוק בפועל:

UI:
- שהמסך נפתח
- שהטבלה מוצגת
- שהטפסים נפתחים
- שהעברית וה-RTL תקינים

Database:
- שטבלת suppliers נוצרה
- ששמירה באמת שומרת
- שעריכה באמת מעדכנת
- שמחיקה באמת מוחקת
- ששליפת הנתונים עובדת

Validation:
- שלא ניתן לשמור בלי שם ספק
- שלא ניתן לשמור בלי מספר ספק
- ששדה אימייל נבדק אם הוזן
- ששדה זמן אספקה מקבל מספר

System:
- שאין שגיאות build
- שאין שגיאות console
- שאין שגיאות database
- שאין שגיאות routing
- שאין שגיאות RTL
