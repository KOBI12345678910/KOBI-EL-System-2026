7. מה כל מודול כזה אמור לעשות
==================================================

אני רוצה שתבין שהמערכת צריכה להיות עצומה ובלתי מוגבלת.

לא מדובר פה על "מסך טפסים".
מדובר על Platform Core שיכול לייצר אינסוף מערכות מתוך עצמו.

לדוגמה:

Module Builder
- יוצר מודולים עסקיים/מערכתיים
- מנהל אייקון, צבע, סדר, תפריט, מצב
- קובע מהו תחום במערכת

Entity Builder
- יוצר אובייקטים עסקיים
- מגדיר סוג ישות
- קושר ישות למודול
- מגדיר behavior flags

Field Builder
- יוצר שדות דינמיים
- תומך בהרבה סוגי שדות
- מגדיר visibility, validation, formatting, formulas

Form Builder
- יוצר טפסים דינמיים
- סקשנים, טאבים, סדר שדות, preview

View Builder
- יוצר רשימות
- עמודות, חיפוש, פילטרים, bulk actions

Detail Page Builder
- בונה כרטיסי ישות
- בלוקים, related data, documents, audit, buttons

Buttons Builder
- מנהל כל כפתור במערכת
- מיקום, צבע, תנאי הצגה, הרשאות, פעולה

Actions Builder
- מגדיר מה כל כפתור עושה
- create/edit/delete/approve/export/custom וכו'

Status Builder
- בונה שכבת סטטוסים חכמה
- צבעים, מעברים, סטטוס ברירת מחדל, סופי

Categories Builder
- בונה קטגוריות והיררכיות
- multi-level taxonomy future-ready

Permissions Builder
- בונה הרשאות לפי מודול/ישות/שדה/פעולה/מסך

Menu Builder
- בונה תפריט מערכת דינמי

Dashboard Builder
- בונה דשבורדים ו-widgets

Workflow Builder
- בונה תהליכים, מעברים, אישורים

Automation Builder
- בונה טריגרים ופעולות אוטומטיות

Template Builder
- מאפשר לשכפל תבניות של מבנים שלמים

Versioning / Publish
- טיוטות, preview, publish, archive, history

Runtime Engine
- קורא metadata
- ומייצר בפועל את המסכים וההתנהגות של המערכת

זה היעד.
לא AI dashboard.
