
📋 רשימת משימות מלאה — טכנו-כל עוזי ERP v1.0.0
כל בעיה + מה בדיוק צריך לעשות — אחד אחד

🔴 משימה #1 — השרת קורס ולא חוזר
בעיה: השרת נופל לחלוטין אחרי כמה דקות שימוש ולא עולה בחזרה לבד. הדף מציג "Run this app to see the results here" — דף ברירת מחדל של Replit כשהאפליקציה כבויה. במהלך הבדיקה השרת קרס ולא חזר כבר 10+ דקות.
מה לעשות:

התקן PM2 או nodemon כ-process manager שיפעיל מחדש את השרת אוטומטית אחרי קריסה
הוסף health check endpoint (לדוגמה /api/health) שבודק שהשרת ו-DB חיים
הוסף watchdog script שבודק כל 30 שניות אם השרת מגיב, ואם לא — מפעיל מחדש
בדוק את לוגים של Replit אחרי קריסה — חפש "out of memory", "SIGKILL", "heap"
הגבל את ה-memory usage של Node.js עם --max-old-space-size=256
שקול לשדרג את תוכנית ה-Replit (יותר RAM + CPU)


🔴 משימה #2 — API endpoints תקועים לנצח
בעיה: כמה API endpoints קריטיים שולחים בקשה ואף פעם לא מקבלים תשובה. הם נשארים ב-status "pending" ללא הגבלת זמן. הנה הרשימה המלאה:

/api/platform/modules — תקוע
/api/platform/modules/21/entities — תקוע
/api/reports-center/executive-dashboard?period=year&year=2026 — תקוע
/api/platform/entities/slug-map — תקוע
/api/payment-anomalies/stats — תקוע
/api/live-ops/history — תקוע
/api/platform/my-permissions — תקוע

מה לעשות:

פתח את הקוד של כל אחד מה-endpoints האלה ובדוק מה הם עושים
הוסף server-side timeout לכל route — אם ה-handler לא סיים תוך 15 שניות, החזר 504 Gateway Timeout
בדוק את ה-database queries מאחורי /api/platform/modules — זה כנראה query כבד שסורק הרבה data. הוסף indexes
בדוק אם יש infinite loop או deadlock בקוד
הוסף logging עם timestamps לכל endpoint כדי לראות איפה זה נתקע
הוסף connection pool limit ל-database (למשל max 10 connections)


🔴 משימה #3 — אותו API נקרא 3 פעמים במקביל
בעיה: הבקשה /api/platform/modules נשלחת 3-4 פעמים בו-זמנית בכל טעינת דף. זה מכפיל את העומס על השרת ומאיץ את הקריסה.
מה לעשות:

חפש בקוד Frontend את כל המקומות שקוראים ל-/api/platform/modules
ודא שיש provider אחד בלבד שטוען את ה-modules (כנראה יש כמה components שכל אחד קורא בנפרד)
אם אתה משתמש ב-React Query / TanStack Query — ודא שה-queryKey זהה בכל המקומות ושה-staleTime מוגדר (לדוגמה 5 דקות)
אם אתה לא משתמש בספריית caching — הוסף אחת, או בנה cache פשוט עם Map שמונע duplicate requests
רכז את כל טעינת ה-platform data ב-context provider אחד ברמת ה-App


🔴 משימה #4 — אין timeout ואין הודעת שגיאה ב-Frontend
בעיה: כשה-API לא מגיב, הדף מציג skeleton loading (אנימציית טעינה) לנצח. אין שום הודעת שגיאה, אין כפתור "נסה שוב", אין שום דבר שאומר למשתמש מה קורה. המשתמש פשוט מחכה ומחכה.
מה לעשות:

הוסף request timeout של 15 שניות לכל API call ב-axios/fetch configuration
הוסף global error interceptor שתופס timeout errors
צור component של ErrorState שמציג: אייקון שגיאה, הודעה "לא הצלחנו לטעון את הנתונים", כפתור "נסה שוב"
בכל דף שיש בו skeleton loading — הוסף timeout: אם עברו 15 שניות ולא נטען, הצג את ErrorState
הוסף React Error Boundary ברמת ה-App שתופס runtime errors ומציג מסך שגיאה ידידותי
הוסף toast/notification כשיש network error


🔴 משימה #5 — Session מתנתק בשקט
בעיה: ה-session של המשתמש פג באמצע שימוש בלי שום התראה. ה-API מחזיר 401 "לא מחובר" אבל ה-UI ממשיך להציג את המערכת כאילו הכל בסדר. המשתמש לא יודע שהוא צריך להתחבר מחדש.
מה לעשות:

הוסף axios interceptor שבודק כל response — אם מקבל 401, עשה redirect לדף login
הצג הודעה למשתמש לפני ה-redirect: "החיבור שלך פג, אנא התחבר מחדש"
הגדל את ה-session TTL בצד השרת (לפחות 8 שעות ליום עבודה)
הוסף session refresh mechanism — כשהמשתמש פעיל, חדש את ה-session אוטומטית
הוסף "heartbeat" שבודק כל 5 דקות אם ה-session עדיין תקף (קריאה ל-/api/auth/me)


🔴 משימה #6 — מסך לבן בטעינה ראשונית
בעיה: כשפותחים כל דף (navigating ישירות ב-URL) — מוצג מסך לבן לחלוטין למשך 5-15 שניות לפני שמשהו מופיע. זה נראה כאילו האתר שבור.
מה לעשות:

הוסף loading indicator ישר ב-index.html בתוך ה-<div id="root"> — ב-CSS טהור, בלי JavaScript. משהו כמו ספינר עם הלוגו של טכנו-כל
עבור מ-Vite dev server ל-production build: npm run build ואז serve את ה-dist folder
הוסף Critical CSS inline ב-head
הוסף font preloading ב-head
הוסף <link rel="preconnect"> לדומיין של ה-API


🟠 משימה #7 — זמן טעינה איטי מאוד (Cold Start)
בעיה: הרצת האפליקציה לוקחת 10-20 שניות. Vite dev server מתחבר מחדש כמה פעמים (ראיתי 3 reconnections בקונסול). בסביבת production זה לא יתקבל.
מה לעשות:

עבור ל-production build: vite build + static file serving
הוסף code splitting — כל מודול (CRM, Finance, HR וכו') צריך להיות lazy loaded
ברוב ה-routes, השתמש ב-React.lazy() + Suspense
הקטן את ה-bundle size — בדוק עם npx vite-bundle-visualizer איזה packages תופסים הכי הרבה
הוסף gzip/brotli compression בצד השרת
הוסף cache headers נכונים לקבצים סטטיים


🟠 משימה #8 — Sidebar navigation לא עובד עקבי
בעיה: לחיצה על פריטים ב-sidebar לפעמים רק גוללת את ה-sidebar ולא מנווטת לדף. לדוגמה: לחצתי על "מחלקת ייצור" (/production/dashboard) — ה-sidebar גלל אבל הדף נשאר דשבורד מנהלים. ה-breadcrumb לא השתנה.
מה לעשות:

בדוק אם ה-sidebar links הם <a href> או <Link> של React Router — ודא שהם <Link> ולא <div onClick>
בדוק אם יש event.stopPropagation() או event.preventDefault() מיותר על ה-sidebar container
בדוק אם יש CSS overflow scroll על ה-sidebar שגורם ל-click events להתבלבל עם scroll events
בדוק שכל פריט ב-sidebar מצביע ל-route נכון ושה-route באמת מוגדר ב-React Router
הוסף visual feedback: כשלוחצים על פריט, צבע אותו מיידית כ-active (גם לפני שהדף נטען)


🟠 משימה #9 — לקוח ללא שם בטבלה
בעיה: בדף ניהול לקוחות (/sales/customers) יש רשומת לקוח CUS-2026-0001 שעמודת "שם" ריקה לגמרי. גם עמודות "טלפון", "אימייל", "עיר" ריקות. הסוג מסומן כ"חברה" והסטטוס "פעיל" והרמה "ברונזה" — אבל אין שם.
מה לעשות:

הוסף validation בטופס יצירת לקוח: שם = שדה חובה, לא ניתן לשמור בלי שם
הוסף validation גם בצד השרת (backend): אם POST/PUT ל-customers בלי שם — החזר 400 Bad Request
תקן את הלקוח הקיים CUS-2026-0001 — מחק אותו או הוסף לו שם
בטבלה: אם שם ריק, הצג placeholder כמו "[ללא שם — יש להשלים]" בצבע אדום
הוסף validation גם ל-טלפון ו-אימייל כשדות מומלצים (לא חובה, אבל עם warning)


🟠 משימה #10 — כרטיסיות ריקות בסקירת פלטפורמה
בעיה: בדף /platform — מתחת לאזור ההתראות של AI יש 4 כרטיסיות שמוצגות כתיבות כהות ריקות לחלוטין. אין בהן טקסט, אין אייקונים, כלום.
מה לעשות:

בדוק את ה-components שאמורים לרנדר בתוך הכרטיסיות האלה — כנראה שהם תלויים ב-API שלא מגיב
הוסף loading state לכל כרטיסייה בנפרד
אם אין data — הצג empty state עם הודעה "אין נתונים" ולא תיבה ריקה
אם הכרטיסיות עדיין לא פותחו — הסתר אותן או הצג "בקרוב"


🟡 משימה #11 — התראת CPU מוצגת כטקסט טכני גולמי
