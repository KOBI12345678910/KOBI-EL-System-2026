אני רוצה שתבנה בתוך המערכת מודול חדש, מרכזי וחשוב מאוד, בשם:

System Builder / Manual Builder / Block Builder

מטרת המודול:
לאפשר לנו לבנות ידנית, מתוך המערכת עצמה, מודולים, טבלאות, שדות, טפסים, תצוגות, תהליכים, דשבורדים ובלוקים — בלי צורך שכל פעם AI יבנה את זה עבורנו.

חשוב מאוד:
זה לא מודול דמה.
אני רוצה מערכת Builder אמיתית, פנימית, שמאפשרת לנו להרחיב את המערכת בעצמנו בצורה ידנית, מסודרת, מאובטחת ומודולרית.

המערכת צריכה להיות בנויה כך שבעתיד נוכל ליצור בעצמנו:
- מחלקות חדשות
- מודולים חדשים
- טבלאות חדשות
- שדות חדשים
- טפסים חדשים
- מסכי רשימה חדשים
- כרטיסי ישות
- אוטומציות
- דשבורדים
- הרשאות
- קשרים בין ישויות

======================================
1. עקרונות בסיס של המודול
======================================

ה-Builder חייב להיות:
- בעברית מלאה RTL
- מודולרי
- מחובר למסד נתונים אמיתי
- מבוסס מטא-דאטה
- ניתן להרחבה
- מאובטח
- מתאים למערכת ERP גדולה
- מתאים לעבודה של עד 200 עובדים
- בנוי כך שלא יהיה צורך לשנות קוד ידנית עבור כל מודול חדש

חשוב:
אני רוצה שהמערכת תאפשר בנייה ידנית דרך ממשק, ולא תדרוש כתיבת קוד בכל פעם.

======================================

======================================

אני רוצה שה-Builder יעבוד בשיטה של metadata-driven system.

כלומר:
במקום שכל מודול יהיה hard-coded,
המערכת תשמור הגדרות בטבלאות מערכת, ומתוכן תייצר את המסכים.

יש ליצור שכבת מערכת עבור:

אם צריך, אפשר להוסיף עוד טבלאות מטא-דאטה תומכות.

4. מודול 1 בתוך ה-Builder: Module Builder
======================================

אני רוצה מסך שבו ניתן ליצור מודול חדש ידנית.

שדות למודול:
- שם מודול בעברית
- שם מודול באנגלית / system key
- תיאור
- קטגוריה
- אייקון
- צבע
- האם פעיל
- סדר בתפריט
- האם להציג בתפריט צד
- האם דורש הרשאות מיוחדות
- הערות מערכת

יכולות:
- יצירת מודול
- עריכת מודול
- ביטול/השבתת מודול
- קביעת מיקום בתפריט
- קביעת אייקון
- קביעת שם תצוגה

דוגמאות למודולים:
- ספקים
- חומרי גלם
- רכש
- קבלני משנה
- ציוד
- רכבים
- עובדים
- פרויקטים

- האם נדרש audit trail
- האם יש soft delete
- האם יש created_at / updated_at
- האם יש owner / responsible user
- האם יש status
- האם יש numbering

יכולות:
- יצירת ישות חדשה
- עריכת ישות
- קישור ישות למודול
- הפעלת/כיבוי ישות
