🔴 משימה #1 — שרת לא יציב (503)
הבעיה: השרת מחזיר שגיאת 503 Service Unavailable באופן אקראי. לפעמים דפים נטענים, לפעמים מסך לבן.
מה לעשות: בדוק את תצורת השרת ב-Replit. בדוק אם יש memory leak או שהשרת מגיע למגבלת זיכרון/CPU. שקול להעביר ל-hosting חזק יותר (Railway, Render, VPS). הוסף health-check endpoint לניטור.

🔴 משימה #2 — Routing שבור (מסך לבן בניווט ישיר)
הבעיה: כשמכניסים URL ישיר כמו /sales/customers או /finance — מופיע מסך לבן. רק ניווט מתוך האפליקציה עובד.
מה לעשות: בקובץ ה-server (server.ts או index.ts), הוסף catch-all route שמחזיר את index.html לכל בקשת GET שלא מתחילה ב-/api/. לדוגמה:
app.get('*', (req, res) => {
  res.sendFile('index.html');
});
זה חייב להיות אחרי כל ה-API routes.

🔴 משימה #3 — דשבורד מנהלים תקוע על Skeleton Loading
הבעיה: דשבורד מנהלים מראה skeleton loading (פלייסהולדרים אפורים) למשך 20-40 שניות ולפעמים לא נטען בכלל.
מה לעשות: 1) הוסף timeout של 10 שניות — אם הנתונים לא הגיעו, הצג הודעת שגיאה עם כפתור "נסה שוב". 2) צמצם את כמות קריאות ה-API שנשלחות בו זמנית בטעינת הדשבורד. 3) הוסף caching בצד הלקוח עם React Query staleTime.

🔴 משימה #4 — סתירת נתונים: דשבורד ראשי מול דשבורד כספים
הבעיה: דשבורד מנהלים מראה הכנסות 0 ₪, הוצאות 0 ₪, רווח 0 ₪. דשבורד כספים מראה הכנסות 3.8M ₪, הוצאות 2.5M ₪, רווח 1.3M ₪. נתונים סותרים לחלוטין.
מה לעשות: פתח את הקוד של דשבורד מנהלים ובדוק מאיזה API endpoint הוא שולף נתונים. השווה ל-API של דשבורד כספים. ככל הנראה דשבורד מנהלים קורא ל-endpoint שבור או שונה. תקן כך ששניהם ישלפו מאותו מקור נתונים.

🔴 משימה #5 — טאבים בדשבורד מראים שגיאת טעינה
הבעיה: לחיצה על טאבים "מכירות", "ייצור", "מלאי", "פרויקטים" בדשבורד הראשי מציגה: "שגיאה בטעינת נתונים — לא ניתן לטעון את נתוני הדשבורד. אנא נסה שוב."
מה לעשות: בדוק את ה-API endpoints שכל טאב קורא. כנראה שה-endpoints לא קיימים או מחזירים שגיאה. צור את ה-endpoints החסרים בשרת או תקן את הקיימים.

🟠 משימה #6 — עמודים ריקים שמופיעים בתפריט
הבעיה: עמודים כמו /platform (סקירת פלטפורמה), /operations-control-center (מרכז בקרה), /crm (דשבורד CRM) מציגים מסך לבן גם כשניגשים אליהם דרך התפריט.
מה לעשות: בדוק ב-App.tsx או בקובץ ה-router אם יש Route מוגדר לכל אחד מהנתיבים האלו. אם חסר — הוסף. אם קיים — בדוק שה-component שמקושר אליו לא זורק שגיאה silent. הוסף Error Boundary לכל עמוד.

🟠 משימה #7 — לקוח נשמר בלי שם (Validation חסר)
הבעיה: אפשר ליצור לקוח חדש (CUS-2026-0001) בלי שם, בלי טלפון, בלי אימייל, בלי כתובת — כל השדות ריקים.
מה לעשות: הוסף validation בטופס יצירת לקוח חדש: שדה "שם לקוח" חייב להיות חובה (required). שדה "טלפון" או "אימייל" — לפחות אחד חובה. מנע שמירה אם השדות החובה ריקים. הצג הודעת שגיאה ברורה.

🟠 משימה #8 — Vite Dev Server בפרודקשן
הבעיה: המערכת רצה על Vite dev server (לא build). כל קובץ .tsx נשלח בנפרד (ראינו 55+ network requests בטעינה אחת), וה-Vite client מנסה reconnect כל הזמן.
מה לעשות: הרץ npm run build (או vite build) כדי ליצור bundle מוכן לפרודקשן. שרת את הקבצים הסטטיים מתיקיית dist. זה יקצר את זמן הטעינה פי 10 לפחות ויפתור הרבה מבעיות היציבות.

🟠 משימה #9 — שגיאת React Key ב-CustomerManagement
הבעיה: בקונסול מופיעה שגיאה: "Each child in a list should have a unique key prop — Check the render method of tbody. It was passed a child from CustomerManagement."
מה לעשות: פתח את הקובץ CustomerManagement.tsx, מצא את הלולאה שמרנדרת שורות בטבלה (map), והוסף prop key ייחודי לכל שורה. לדוגמה: <tr key={customer.id}> במקום <tr>.

🟡 משימה #10 — גרפים ריקים בדשבורד (סקירה כללית)
הבעיה: 4 גרפים בטאב "סקירה כללית": הכנסות מול הוצאות, תזרים מזומנים, פילוח הוצאות, ביצועי מחלקות — כולם ריקים עם הודעת "אין נתונים".
מה לעשות: קשור למשימה #4. לאחר תיקון מקור הנתונים, הגרפים צריכים להתמלא. בנוסף, הוסף empty state יפה עם הסבר ו-call-to-action (למשל: "אין נתונים עדיין. לחץ כאן להוספת נתונים").

🟡 משימה #11 — כפתור "פו טיפ" לא עובד
הבעיה: בעמוד ניהול לקוחות, ברשומת לקוח מורחבת יש כפתור "פו טיפ" שלחיצה עליו לא עושה כלום.
מה לעשות: בדוק את הקוד של הכפתור — ייתכן שחסר onClick handler. אם הפיצ'ר לא מוכן — הסתר את הכפתור או הוסף tooltip "בקרוב". אם מוכן — חבר אותו לפעולה (פתיחת tooltip, modal, או ניווט).

🟡 משימה #12 — כפילות עמודים בתפריט
הבעיה: יש דפים כפולים: "מוצרים מוגמרים" + "מוצרים מוגמרים (ישות)", "מלאי חומרי גלם" + "מלאי חומרי גלם (ישות)", "ניהול מלאי" + "ניהול מלאי (כללי)", "דשבורד מלאי" + "דשבורד מלאי (מתקדם)".
מה לעשות: החלט אם צריך שתי גרסאות. אם כן — שנה שמות כדי שההבדל יהיה ברור. אם לא — מחק את הכפילויות מהתפריט. עמודי "(ישות)" נראים כעמודי builder/debug — הסר מהתפריט למשתמש רגיל.

🟡 משימה #13 — תפריט צד מנופח מדי
הבעיה: תפריט צד עם מאות פריטים (כספים: 124, ייצור: 74, לקוחות: 72 וכו'). המשתמש צריך לגלול הרבה ומתבלבל.
מה לעשות: 1) הגבל את התפריט ל-3-5 פריטים ראשיים בכל קטגוריה, עם כפתור "הצג הכל". 2) הסתר עמודים שלא רלוונטיים לתפקיד המשתמש. 3) הוסף חיפוש בתפריט. 4) הסר עמודי debug/builder מהתפריט.

🟡 משימה #14 — אין Splash Screen / Loading מיידי
הבעיה: בטעינה ראשונה של האפליקציה יש כמה שניות של מסך לבן לפני שנראה משהו.
מה לעשות: הוסף CSS inline ב-index.html עם לוגו ואנימציית loading שמופיעים מיד, לפני שה-JavaScript נטען. הסר אותם ב-useEffect אחרי שהאפליקציה מוכנה.

🔵 משימה #15 — מספרים ליד מודולים בתפריט לא ברורים
הבעיה: ליד כל מודול בתפריט מופיע מספר (כספים: 124, ייצור: 74). לא ברור מה המספר מייצג.
מה לעשות: הוסף tooltip שמסביר (למשל: "124 תתי-עמודים"). אם המספר לא מועיל — הסר אותו.

🔵 משימה #16 — צבע רמת לקוח "ברונזה" מטעה
הבעיה: רמת לקוח "ברונזה" מוצגת בירוק, מה שנראה חיובי. ברונזה היא הרמה הנמוכה.
מה לעשות: שנה את הצבע לחום/ברונזה או אפור. ירוק שמור לזהב/פלטינה. כתום לכסף. חום לברונזה.

🔵 משימה #17 — בדוק שכל עמודי התפריט עובדים
הבעיה: עשרות עמודים בתפריט שלא ניתן היה לבדוק בגלל זמני הטעינה הארוכים.
מה לעשות: אחרי תיקון משימות 1-2-8, עבור על כל עמוד בתפריט ובדוק: האם הוא נטען? האם יש תוכן? האם הכפתורים עובדים? רשום כל עמוד שבור ברשימה נפרדת.

🔵 משימה #18 — הוסף Error Boundaries
הבעיה: אם קומפוננט זורק שגיאה, כל העמוד קורס למסך לבן בלי הסבר.
מה לעשות: עטוף כל עמוד ראשי ב-React Error Boundary שמציג הודעה ידידותית: "משהו השתבש. לחץ כאן לרענון" במקום מסך לבן.
