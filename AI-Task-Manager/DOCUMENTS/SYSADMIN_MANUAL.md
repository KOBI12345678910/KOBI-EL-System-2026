## 2. גיבוי ושחזור

### גיבוי מסד הנתונים

#### גיבוי אוטומטי
Replit מבצע גיבויים אוטומטיים של מסד הנתונים. ניתן לראות את הגיבויים הזמינים בממשק Replit.

#### גיבוי ידני
להרצת גיבוי ידני:

```bash
# גיבוי לקובץ SQL
pg_dump $DATABASE_URL > backup_$(date +%Y%m%d_%H%M%S).sql

# דחיסת הקובץ
gzip backup_*.sql
```

#### לו"ז גיבויים מומלץ
- **גיבוי יומי:** כל לילה בשעה 02:00
- **גיבוי שבועי:** כל ראשון בשעה 01:00
- **גיבוי חודשי:** בתחילת כל חודש

#### אחסון גיבויים
- שמור גיבויים יומיים ל-30 יום
- שמור גיבויים שבועיים ל-3 חודשים
- שמור גיבויים חודשיים ל-12 חודשים

### שחזור מסד הנתונים

**אזהרה: שחזור מסד נתונים ימחק את כל הנתונים הנוכחיים!**

1. עצור את כל ה-Workflows לפני השחזור
2. בצע גיבוי של הנתונים הנוכחיים (להיות בטוח)
3. הרץ את פקודת השחזור:

```bash
# שחזור מקובץ SQL
psql $DATABASE_URL < backup_YYYYMMDD_HHMMSS.sql
```

4. בדוק את שלמות הנתונים לאחר השחזור
5. הפעל מחדש את ה-Workflows

### גיבוי קוד המערכת
הקוד מנוהל על ידי Replit עם checkpoints אוטומטיים. לשחזור לגרסה קודמת:
1. בממשק Replit — לחץ על "History"
2. בחר את ה-checkpoint הרצוי
3. לחץ "Restore"

## 3. ניהול משתמשים

### הוספת משתמש חדש

1. היכנס למערכת עם הרשאות Admin
2. לחץ **"הגדרות"** → **"ניהול משתמשים"**
3. לחץ **"+ הוסף משתמש"**
4. מלא:
   - **שם מלא**
   - **כתובת מייל** (תשמש כשם משתמש)
   - **תפקיד** (ראה סוגי תפקידים למטה)
   - **מחלקה** (אם רלוונטי)
5. לחץ **"שמור"** — המשתמש יקבל מייל עם הוראות כניסה

### סוגי תפקידים

| תפקיד | גישה | מתאים ל |
|--------|------|---------|
| **Admin** | גישה מלאה לכל המערכת | קובי |
| **מנהל** | גישה לרוב המודולים, ניהול עובדים | דימה, עוזי |
| **עובד** | גישה לנוכחות, משימות, חופשות | עובדי מפעל |
| **קריאה בלבד** | צפייה בלבד, ללא עריכה | מבקרים, מנהלים חיצוניים |
| **חשב** | גישה מלאה למודולי פיננסים | רו"ח |

### איפוס סיסמה
1. מצא את המשתמש ברשימת המשתמשים
2. לחץ על שם המשתמש
3. לחץ **"אפס סיסמה"**
4. בחר **"שלח למייל"** — המשתמש יקבל קישור לאיפוס

### השבתת משתמש (עזב את החברה)
1. מצא את המשתמש
2. לחץ על שם המשתמש
3. לחץ **"השבת"** (לא "מחק" — שמור על ההיסטוריה)
4. אשר את הפעולה

**חשוב:** לעולם אל תמחק משתמש — רק השבת אותו. כל הפעולות שביצע ישמרו בלוג.

### ביקורת גישה
מדי חודש בדוק:
- האם יש משתמשים לא פעילים שצריך להשבית
- האם הרשאות כולם מתאימות לתפקידם
- לחץ **"הגדרות"** → **"יומן ביקורת"** לראות פעילות חשודה

## 4. הגדרות מערכת

### הגדרות כלליות

| הגדרה | נתיב | הערות |
|-------|------|-------|
| פרטי חברה | הגדרות → כללי | שם, כתובת, לוגו |
| מע"מ ומיסים | הגדרות → פיננסים | שיעור מע"מ נוכחי |
| מסמכי מס | הגדרות → פיננסים → ישראל | אינטגרציות מס ישראליות |
| לוגיקת שכר | הגדרות → שכר | חוקי עבודה ישראלים |
| שעות עבודה | הגדרות → כ"א | שעות יום עבודה |

### הגדרות אינטגרציות

**מייל (דואר אלקטרוני):**
- נתיב: הגדרות → אינטגרציות → מייל
- ספק מייל ופרטי SMTP

**WhatsApp/SMS:**
- נתיב: הגדרות → אינטגרציות → WhatsApp
- נדרש API key מהספק

### עדכוני מערכת

כאשר יש עדכון זמין:
1. תוצג הודעה בממשק Replit
2. בחר שעה שקטה לעדכון (בדרך כלל סוף שבוע)
3. גבה את מסד הנתונים לפני העדכון
4. בצע את העדכון בממשק Replit
5. בדוק שכל המודולים פועלים לאחר העדכון

## 5. פתרון תקלות נפוצות

### המערכת לא נטענת / שגיאה 500

**שלבי בירור:**
1. בדוק שה-Workflows פועלים (API Server + ERP App)
2. צפה ב-Logs של ה-Workflow לאיתור שגיאות
3. בדוק שמסד הנתונים זמין

**פתרון:**
```bash
# הפעל מחדש את שרת ה-API
# (דרך ממשק Replit — לחץ Restart על Workflow)
```

### משתמש לא יכול להתחבר

**שלבי בירור:**
1. בדוק שחשבון המשתמש פעיל (לא מושבת)
2. בדוק שהסיסמה נכונה — אפס אם צריך
3. בדוק אם יש נעילת חשבון (ניסיונות כניסה כושלים רבים)

**פתרון:** אפס סיסמה ושחרר נעילה מהגדרות המשתמש.

### נתונים לא מתעדכנים / שמירה לא עובדת

**שלבי בירור:**
1. בדוק ב-Network tab של הדפדפן אם קריאות ה-API מצליחות
2. בדוק logs של שרת ה-API

**פתרון נפוץ:** בדרך כלל בעיית הרשאות — בדוק שלמשתמש יש הרשאות מתאימות.

### ביצועים איטיים

**שלבי בירור:**
1. בדוק כמות משתמשים מחוברים בו-זמנית
2. בדוק queries איטיים בלוג מסד הנתונים

**פתרון:**
- ניקוי cache
- עדכון אינדקסים במסד הנתונים (צור קשר עם ספק המערכת)

### שגיאות בחישוב שכר

**שלבי בירור:**
1. בדוק שנתוני הנוכחות מלאים לתקופה
2. בדוק שהגדרות השכר של העובד מעודכנות
3. בדוק שהגדרות מס מעודכנות (שנת מס נוכחית)

**פתרון:** תקן נתונים שגויים ורוץ את השכר מחדש.

### שגיאות מסד נתונים

**הודעות שגיאה נפוצות:**

| שגיאה | משמעות | פתרון |
|-------|---------|-------|
| `Connection timeout` | בעיית חיבור ל-DB | הפעל מחדש שרת API |
| `Unique constraint` | ערך כפול בשדה ייחודי | בדוק אם הרשומה קיימת |
| `Foreign key violation` | מחיקת רשומה בשימוש | ודא שאין תלויות |
