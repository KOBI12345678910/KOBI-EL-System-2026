## 4. שלב ג' — שבוע ראשון (הרצה מקבילה)

> **הרצה מקבילה:** חלק מהנתונים מוזנים גם במערכת הישנה וגם החדשה לצורך השוואה.

### כללים להרצה מקבילה

1. **מה מוזן בשתי מערכות:** חשבוניות, הזמנות עבודה, שכר
2. **מה רק בחדשה:** נוכחות, בקשות חופשה, תקשורת פנימית
3. **בדיקה יומית:** עוזי משווה סיכומי יום בין המערכות

### לוח מעקב שבועי

| יום | פעולה | אחראי | ✓ |
|---|---|---|---|
| ראשון | בדיקת נוכחות — מספר עובדים מחוברים | IT | [ ] |
| שני | השוואת חשבוניות — מערכת ישנה vs חדשה | עוזי | [ ] |
| שלישי | בדיקת שכר — חישובים תקינים | עוזי | [ ] |
| רביעי | ישיבת סטטוס שבועית | קובי + כולם | [ ] |
| חמישי | בדיקת ביצועי שרת (עומסי שיא) | IT | [ ] |
| שישי | גיבוי שבועי + דוח סיכום שבוע | IT + קובי | [ ] |

### קריטריון לסגירת הרצה מקבילה

- [ ] פחות מ-3 שגיאות קריטיות בשבוע
- [ ] עוזי מאשר שהמספרים תואמים
- [ ] כל העובדים משתמשים במערכת החדשה באופן עצמאי

## 6. תוכנית Rollback

### מתי להפעיל Rollback

1. **מיידי (תוך 2 שעות):** המערכת לא זמינה ליותר מ-50% מהמשתמשים
2. **ביום הראשון (17:00):** יותר מ-5 שגיאות קריטיות בלתי פתירות
3. **בשבוע הראשון:** נתונים פגומים שלא ניתן לתקן

### נוהל Rollback

```
שלב 1: קובי מאשר החלטת Rollback
שלב 2: IT מודיע לכל המשתמשים ל"הפסיק עבודה"
שלב 3: IT מוציא המערכת החדשה למצב תחזוקה
שלב 4: IT משחזר גיבוי אחרון (30-60 דקות)
שלב 5: IT בודק תקינות המערכת הישנה
שלב 6: קובי מאשר חזרה לעבודה
שלב 7: תיעוד גורם הכשל ותכנון תיקון
```

### אחסון גיבויים

- **מיקום גיבוי:** שרת IT + ענן (S3)
- **גיבוי לפני Go-Live:** חובה
- **גיבויים יומיים בשבוע ראשון:** 06:00, 12:00, 18:00
