## כללי (General)

### 1. מה זו המערכת ה"מגה-ERP" שלי?
חבילה של 5 פרויקטים משולבים: **onyx-procurement** (רכש), **payroll-autonomous** (שכר), **techno-kol-ops** (תפעול מפעל בזמן אמת), **onyx-ai** (AI למס ומע"מ), ו-**client** (ממשק משתמש). יחד הם מהווים מערכת ERP מלאה לעסק קטן-בינוני (Techno-Kol Uzi).

### 2. האם אני צריך חיבור אינטרנט קבוע?
לא. המערכת פועלת מקומית (Offline-first). חיבור נדרש **רק** כאשר אתה מגיש PCN836 לרשות המסים, מקצה מספרי חשבונית (רפורמת 2024), שולח וואטסאפ, או מסנכרן עם הבנק.

### 3. היכן הנתונים שלי נשמרים?
במסד נתונים מקומי (PostgreSQL או SQLite) בתיקיית הפרויקט. גיבוי יומי אוטומטי ל-`./backups/`. מומלץ להוסיף גיבוי חיצוני (OneDrive / כונן USB מוצפן).

### 4. האם המערכת תואמת חוק?
כן. המערכת תואמת את:
- **חוק הגנת השכר תיקון 24** (תלושי שכר עם 17+ שדות חובה).
- **חוק מע"מ** (שמירת ספרים 7 שנים, PCN836).
- **רפורמת חשבונית ישראל 2024** (הקצאת מספרי חשבונית לחשבוניות > 25,000 ₪).

### 5. מה אני עושה אם המערכת נופלת?
1. שמור על רגיעה — הנתונים שלך בטוחים ב-DB וב-backups.
2. הפעל מחדש את השרת (`npm run dev` / `python manage.py runserver`).
3. אם התקלה חוזרת — ראה RUNBOOK (Placeholder: `./RUNBOOK.md`).

## רכש (Procurement)

### 6. איך אני שולח בקשת מחיר לכמה ספקים בבת אחת?
**רכש → "+ בקשת מחיר חדשה"** → בחר מספר ספקים מהרשימה (Ctrl+Click) → לחץ "שלח RFQ". המערכת שולחת לכולם במקביל.

### 7. איך המערכת משווה הצעות באופן אוטומטי?
המערכת מחשבת ציון (score) לכל הצעה לפי: **מחיר (50%) + זמן אספקה (25%) + אמינות היסטורית של הספק (25%)**. הציון מוצג בטבלת ההשוואה.

### 8. האם אני יכול לשלוח PO בוואטסאפ?
כן. אחרי הפקת ה-PO, לחץ **"שלח בוואטסאפ"**, בחר את איש הקשר אצל הספק, וה-PDF נשלח אוטומטית דרך WhatsApp Business API.

### 9. איך אני מוסיף ספק חדש?
**רכש → ספקים → "+ ספק חדש"**. מלא: שם, ח.פ., טלפון, אימייל, תחום (מתכת / חומרים / שירותים).

### 10. איך אני מבטל RFQ?
פתח את ה-RFQ מהרשימה, לחץ **"בטל"**. הסטטוס משתנה ל-"מבוטל" אבל הרשומה לא נמחקת (לצורך ביקורת).

## מע"מ (VAT)

### 11. איך אני מפיק PCN836?
**מע"מ → סגירת תקופה** → בחר חודש → לחץ **"הפק PCN836"** → לחץ **"הגש לרשות המסים"** (או הורד ידנית והעלה לאתר שע"מ).

### 12. מה זה מספר הקצאה (Allocation Number)?
החל מרפורמת 2024, חשבוניות מעל **25,000 ₪** דורשות מספר הקצאה מרשות המסים. המערכת מבקשת אותו אוטומטית מה-SHAAM API בזמן הפקת החשבונית.

### 13. מה אם שכחתי להזין חשבונית?
אתה יכול להוסיף אותה בדיעבד **לפני** סגירת התקופה. אחרי סגירה — יש צורך בתיקון PCN836 (פעולה מיוחדת — ראה Support).

### 14. האם המערכת מזהה חשבונית סרוקה (OCR)?
כן (במודול onyx-ai). העלה PDF / JPG → המערכת תקלוט: ח.פ. ספק, סכום, מע"מ, תאריך. ראה "חשבוניות → יבוא OCR".

### 15. מה שיעור המע"מ הנוכחי?
**17%** (2026). ניתן לעדכן ב-**הגדרות → פרופיל עוסק → שיעור מע"מ**.

## מס שנתי (Annual Tax)

### 16. מתי המערכת מייצרת טופס 1301?
בכל זמן אחרי סגירת שנת מס. **מס שנתי → סגירת שנה** → בחר שנה → לחץ **"הפק 1301"**.

### 17. מה ההבדל בין 1301 ל-1320 ל-6111?
- **1301** — דוח שנתי ליחיד / עצמאי.
- **1320** — נספח לדוח 1301 (הכנסות מיגיעה אישית / מעבודה).
- **6111** — דוח מאוחד (חברות / שותפויות).

### 18. איך אני מוסיף פרויקט נדל"ן?
**מס שנתי → פרויקטים → "+ פרויקט חדש"** → בחר סוג "נדל"ן" → מלא כתובת נכס, תאריך רכישה, עלות רכישה.

### 19. איך המערכת מחשבת פחת?
אוטומטית לפי סוג הנכס וכללי רשות המסים (קווי / מואץ). ניתן לראות בדוח **"פחת ונכסים"** במודול מס שנתי.

## בנק (Bank)

### 20. אילו פורמטים של דף חשבון המערכת תומכת?
**CSV** (כל בנק ישראלי) ו-**MT940** (סטנדרט בינלאומי). פועלים, לאומי, דיסקונט, מזרחי-טפחות — כולם נתמכים ב-CSV.

### 21. מה זו "התאמה אוטומטית"?
המערכת משווה טרנזקציות בבנק לחשבוניות/תקבולים במערכת. התאמה נעשית לפי: **סכום + תאריך (±3 ימים) + רמז בהערות**. ציון אמינות 90%+ → סימון אוטומטי.

### 22. מה אם ההתאמה שגויה?
לחץ **"דחה"** על ההתאמה המוצעת, ואז **"התאם ידנית"** כדי לבחור את החשבונית הנכונה. המערכת לומדת מהתיקון שלך.

### 23. איך אני מייבא דף חשבון?
**בנק → יבוא** → גרור קובץ CSV / MT940 → בחר חשבון → "יבא". המערכת מזהה פורמט אוטומטית.

## שכר (Payroll)

### 24. איך אני מחשב תלוש חודשי לכל העובדים?
**שכר → חישוב חודשי** → בחר חודש → לחץ **"חשב הכל"**. המערכת מחשבת ברוטו, ניכויים (מס הכנסה, ביטוח לאומי, בריאות, פנסיה), ונטו.

### 25. מה השדות החובה לפי חוק הגנת השכר תיקון 24?
17 שדות, כולל: שם עובד, תפקיד, שכר בסיס, שעות עבודה רגילות, שעות נוספות, ניכויים מפורטים, הפקדות לקרן פנסיה, ימי חופשה/מחלה שנצברו ונוצלו, סך נטו, חתימה דיגיטלית. המערכת מטפלת בכולם אוטומטית.

### 26. איך אני מוסיף עובד חדש באמצע חודש?
**שכר → עובדים → "+ עובד חדש"** → הזן תאריך התחלה אמיתי → המערכת מחשבת חלקית אוטומטית לחודש הראשון (Pro-rata).

### 27. איך אני מתקן תלוש שהונפק בטעות?
- **לפני אישור:** פתח את התלוש ולחץ "ערוך".
- **אחרי אישור:** הנפק **"תלוש מתקן"** (תלוש חדש שמבטל את הקודם) — במודול → "תלוש מתקן".

### 28. האם המערכת שולחת תלוש לעובד אוטומטית?
כן, אם הגדרת אימייל / מספר וואטסאפ בפרטי העובד. לחץ **"שלח תלוש לעובד"** → התלוש (PDF מוצפן) נשלח.

## טכני ואבטחה (Technical & Security)

### 29. איך אני מגבה את הנתונים שלי?
גיבוי יומי אוטומטי ל-`./backups/YYYY-MM-DD/`. בנוסף, לחץ **"הגדרות → גיבוי ידני"** בכל זמן. מומלץ להעתיק ל-USB מוצפן או OneDrive מדי שבוע.

### 30. אני חושד שמישהו נכנס לחשבון שלי — מה אני עושה?
1. **הגדרות → אבטחה → "החלף סיסמה"** מיד.
2. לחץ **"רענן את כל המפתחות"** — כל ה-API keys מתחלפים (SHAAM, WhatsApp, Bank).
3. סקור את ה-Audit Log (**הגדרות → Audit Log**) לפעולות חשודות.
4. אם יש חשד לגניבת נתוני רשות המסים — דווח לשע"מ מיד.
