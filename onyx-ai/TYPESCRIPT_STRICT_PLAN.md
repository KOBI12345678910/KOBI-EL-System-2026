| 1 | `noImplicitAny`                  | yes                       | **OFF (explicitly disabled)** |
| 2 | `strictNullChecks`               | yes                       | ON              |
| 3 | `strictFunctionTypes`            | yes                       | ON              |
| 4 | `strictBindCallApply`            | yes                       | ON              |
| 5 | `strictPropertyInitialization`   | yes                       | ON              |
| 6 | `alwaysStrict`                   | yes                       | ON              |
| 7 | `useUnknownInCatchVariables`     | yes                       | ON              |
| 8 | `noImplicitThis`                 | yes                       | ON              |

### Notable specific patterns

**`integrations.ts`** ‚Äî the HTTP/logger infrastructure:
```ts
// lines 32-35  (ILogger)
debug(msg: string, meta?: any): void;
info(msg: string, meta?: any): void;
warn(msg: string, meta?: any): void;
error(msg: string, meta?: any): void;

// line 52  (ToolConfig)
execute: (input: any, ctx?: ToolContext) => Promise<Result<any, Error>>;

// line 74
body?: any;              // HttpOptions

// line 178
} catch (e: any) { ... }

// line 192-198
async post<T = any>(url: string, body?: any, opts = {}) { ... }
```
*Fix strategy:* replace with `unknown`, generics, or `Record<string, unknown>`.
Most of these are safe to widen to `unknown` because the values are forwarded,
not dereferenced, inside the function body.

**`onyx-integrations.ts`** ‚Äî HTTP response bodies:
```ts
// repeats 10+ times
const body = res.body as any;
```
*Fix strategy:* declare per-integration response interfaces OR use
`const body = res.body as Record<string, unknown>;` and parse fields with
type guards. This is the single largest mechanical change.

**`index.ts` / `onyx-platform.ts`** ‚Äî `Map.get` with `!`:
```ts
this.subscribers.get(eventType)!.push(handler);
this.typeIndex.get(entity.type)!.add(entity.id);
this.tagIndex.get(tag)!.add(entity.id);
```
*Fix strategy:* introduce a `getOrCreate` helper:
```ts
function getOrCreate<K, V>(map: Map<K, V>, key: K, factory: () => V): V {
  const existing = map.get(key);
  if (existing) return existing;
  const fresh = factory();
  map.set(key, fresh);
  return fresh;
}
```
These break under `noUncheckedIndexedAccess` even though they currently pass.

## 4. Recommended migration sequence

Incremental approach. Each step is a **separate PR** with `npm run typecheck`
passing before merge. Run in order ‚Äî later steps build on earlier invariants.

### Step 1 ‚Äî Enable `noImplicitAny` (MEDIUM)

**Effort: M (1‚Äì2 days)**
**Risk: Low** ‚Äî only flags *missing* annotations, not explicit `any`.

Most function params and returns are already annotated. Expected failures:
a few unannotated closure callbacks inside `integrations.ts` / `onyx-integrations.ts`
(lambda args like `(b) => ...` or `find(h => ...)` where the array is already `any`).

Steps:
1. Remove `"noImplicitAny": false` from `tsconfig.json`.
2. Run `npx tsc --noEmit` ‚Äî expect 20‚Äì40 errors, mostly in the two integration files.
3. Annotate the flagged callbacks with explicit `: unknown` or proper types.
4. CI: add `npm run typecheck` to pre-commit.

**SAFE. Do first.**

### Step 2 ‚Äî Replace gratuitous `: any` with `unknown` (LARGE)

**Effort: L (3‚Äì5 days)**
**Risk: Medium** ‚Äî may surface real type holes.

Target the 124 `any` occurrences, prioritized by file:
1. `onyx-integrations.ts` (58) ‚Äî batch-replace `as any` ‚Üí `as Record<string, unknown>`, then tighten case-by-case.
2. `integrations.ts` (54) ‚Äî change `ILogger.meta: any` ‚Üí `meta?: unknown`, `HttpOptions.body: any` ‚Üí `body?: unknown`, generic `<T = any>` ‚Üí `<T = unknown>`.
3. `onyx-platform.ts` / `index.ts` (5 each) ‚Äî trivial.
4. Module files (3 `as any` total) ‚Äî trivial.

This step does *not* require a tsconfig change. You can do it under the current
config, then later strict flags will simply *verify* the work.

**Commits should be file-scoped** so reviewers can sanity-check one integration at a time.

### Step 3 ‚Äî Enable `noUnusedLocals` + `noUnusedParameters` (SMALL)

**Effort: S (half day)**
**Risk: None**

Add:
```jsonc
"noUnusedLocals": true,
"noUnusedParameters": true
```

Clean up dead imports and unused catch bindings. Usually hundreds of tiny mechanical fixes. Agent-safe.

### Step 4 ‚Äî Enable `noImplicitOverride` (SMALL)

**Effort: S (half day)**
**Risk: Low**

Adds `override` keyword requirement on subclass methods. The codebase uses classes
(e.g. `ConsoleLogger implements ILogger`) but very little inheritance, so small impact.

### Step 5 ‚Äî Enable `exactOptionalPropertyTypes` (MEDIUM)

**Effort: M (1‚Äì2 days)**
**Risk: Medium** ‚Äî will flag `{ foo: undefined }` vs `{}` distinctions.

Common failure: `value: x ?? undefined` when the interface says `value?: T`.
Look at the 167 `undefined` usages to estimate; most are legitimate value checks.

### Step 6 ‚Äî Enable `noUncheckedIndexedAccess` (LARGE)

**Effort: L (3‚Äì5 days)**
**Risk: High** ‚Äî biggest behavior-adjacent change in the whole migration.

This makes `arr[i]` and `map.get(k)` return `T | undefined`. The 21 non-null
assertions will all need to become explicit checks or use the `getOrCreate`
helper pattern above. Do this AFTER step 1‚Äì2 so you're not fighting `any` holes
at the same time.

Most module files have zero non-null assertions, so this will hit the core
event-sourcing / knowledge graph files (`index.ts`, `onyx-platform.ts`)
almost exclusively.

### Step 7 ‚Äî Enable `noPropertyAccessFromIndexSignature` (SMALL)

**Effort: S (half day)**
**Risk: Low**

Forces `obj["key"]` instead of `obj.key` for index-signature types. The main
offenders will be HTTP response bodies and `Record<string, unknown>` patterns
introduced in Step 2.

## 5. Effort summary and SAFE vs RISKY matrix

| Step | Flag(s)                            | Effort | Risk   | Safe to run on its own? |
|-----:|-------------------------------------|--------|--------|-------------------------|
| 1    | `noImplicitAny`                     | M      | Low    | Yes                     |
| 2    | Replace `any` with `unknown` (no flag change) | L | Medium | Yes                 |
| 3    | `noUnusedLocals`, `noUnusedParameters` | S   | None   | Yes                     |
| 4    | `noImplicitOverride`                | S      | Low    | Yes                     |
| 5    | `exactOptionalPropertyTypes`        | M      | Medium | Yes                     |
| 6    | `noUncheckedIndexedAccess`          | L      | High   | Only after 1 and 2      |
| 7    | `noPropertyAccessFromIndexSignature`| S      | Low    | After 2                 |

**Total effort estimate: 2‚Äì3 engineer-weeks** for the full migration on this
codebase, assuming one engineer familiar with the domain.

### Absolute SAFE set (green, do any time)
- Step 3 (unused locals/params)
- Step 4 (override)
- Step 7 (index signature access)

### Medium risk (gated on tests / manual QA)
- Step 1 (`noImplicitAny`) ‚Äî low code risk, but changes the default contract.
- Step 2 (`any` ‚Üí `unknown`) ‚Äî biggest churn, reveals real bugs.
- Step 5 (`exactOptionalPropertyTypes`) ‚Äî subtle runtime semantics.

### RISKY (requires a test suite first)
- Step 6 (`noUncheckedIndexedAccess`) ‚Äî will force behavior changes around
  `Map.get().foo` patterns. **Do NOT attempt without at least smoke tests.**
  The codebase currently has `"test": "echo \"tests coming soon\" && exit 0"`,
  so this step should be blocked on real tests being added.
