## 2. ממצאי ליבה — קבועי פנסיה ופיצויים

### 2.1 קבועים בקוד (App.jsx שורה 11)

```js
const PEN_E=.06, PEN_R=.065, SEV=.0833, VAT=.17, WORK_DAYS=22, WORK_HRS=186;
```

| קבוע | ערך | משמעות | נדרש לפי דין |
|------|-----|--------|----------------|
| `PEN_E` | 6.0% | הפרשת עובד לפנסיה | 6.0% (תואם צו הרחבה) |
| `PEN_R` | 6.5% | הפרשת מעסיק לתגמולים | 6.5% (תואם צו הרחבה) |
| `SEV`   | 8.33% | רכיב פיצויים | 8.33% = 1/12 (תואם חוק פיצויי פיטורין) |

### 2.2 חישוב הפרשות (שורות 59–64)

```js
const penE = emp.hasPension ? gross * PEN_E : 0;
const penR = emp.hasPension ? gross * PEN_R : 0;
const sev  = gross * SEV;
const totalDeductions = tax + bl + ht + penE;
const net = gross - totalDeductions;
const erCost = gross + penR + sev + bl*1.05;
```

**תצפית חיובית:** שיעורי ההפרשה (6% / 6.5% / 8.33%) תואמים במדויק את דרישות צו ההרחבה לפנסיית חובה.

## 3. בדיקה מול שמונה דרישות ה־QA

### דרישה 1 — פנסיה חובה לפי צו הרחבה
**סטטוס:** חלקי (PARTIAL)

- הקוד מציע **checkbox ידני** לכל עובד: `hasPension` (שורות 59, 521).
- **אין אכיפה** של חובת הפרשה כצו הרחבה דורש (עובד 21+ עם 6 חודשי ותק, או שכיר עם קרן פעילה מיום 1).
- **חסר:** בדיקת גיל, ותק, וחריגות (עובד זר, בעל שליטה, נער עובד).
- **סיכון:** אפשר לסמן `hasPension=false` לעובד שחייב בפנסיה בחוק = חשיפה משפטית.

### דרישה 2 — הפרשות עובד 6% + מעסיק 6.5% + פיצויים 6%
**סטטוס:** חלקי (PARTIAL)

- 6% עובד — תקין (`PEN_E=.06`).
- 6.5% מעסיק — תקין (`PEN_R=.065`).
- פיצויים — **לא תואם דרישת ה־QA!** הקוד משתמש ב־`SEV=.0833` (8.33% = 1/12) ולא 6%.
  - **הערה:** 8.33% הוא השיעור המלא לפיצויי פיטורין לפי סעיף 14.
  - **הערה:** 6% הוא השיעור המינימלי בצו הרחבה לפנסיה חובה.
  - **פער:** המערכת אינה מבחינה בין שני המצבים ואינה מאפשרת בחירה (סעיף 14 מלא / חלקי / ללא).

### דרישה 3 — תקרת שכר להפרשות (פעמיים שכר ממוצע במשק)
**סטטוס:** חסר לחלוטין (MISSING)

- **אין שום התייחסות** לתקרת הפרשות לפנסיה בקוד.
- `penE = gross * PEN_E` — מוחל על **כל** הברוטו ללא `Math.min(gross, CAP)`.
- **חסר קבוע:** `PEN_CAP ≈ 24,730 ₪` (פעמיים שכר ממוצע 2025).
- **השלכה:** עבור עובדים בשכר גבוה, המערכת מחשבת הפרשות עודפות ומתעלמת מתקרת הזיכוי/חבות המס הרלוונטית.
- **השוואה לקבועי BL:** עבור ביטוח לאומי יש תקרה (`BL_T=7122`), אך עבור פנסיה — אין.

### דרישה 4 — סוגי קרנות (פנסיה / קופ"ג / ביטוח מנהלים)
**סטטוס:** חסר לחלוטין (MISSING)

- `hasPension` הוא `boolean` בינארי בלבד. אין שדה `fundType`, `fundId`, `fundName`.
- **אין תמיכה** בהבחנה בין:
  - קרן פנסיה מקיפה (חדשה/ברירת מחדל)
  - קופת גמל לקצבה
  - ביטוח מנהלים (פוליסת ביטוח חיים משתתפת ברווחים)
  - קרן פנסיה ותיקה
- **חסר:** שדה ספק (מנורה, מגדל, כלל, הראל וכו'), מספר פוליסה, תאריך הצטרפות.

### דרישה 5 — חלוקת הפרשות בין קרנות
**סטטוס:** חסר לחלוטין (MISSING)

- כל עובד יכול להחזיק בקרן/קופה אחת בלבד מכוח המודל.
- **חסר:** אפשרות לחלוקת ברוטו בין מספר קרנות (למשל 5% לקרן פנסיה + 1% לקופ"ג + 2.5% ביטוח מנהלים).
- **חסר:** טיפול במצב של "קרן בחירה" (תעשיה, בנייה, חקלאות).
- **חסר:** מנגנון ransfer בין קרנות (ניוד).

### דרישה 6 — הטבת מס (זיכוי/ניכוי 35%/5%)

## 5. ממצאי BUG / פערים טכניים

### BUG-PEN-01 (בינוני)
**מיקום:** `src\App.jsx:61`
**תיאור:** `const sev = gross * SEV;` — הפיצויים נצברים תמיד, אפילו כאשר `hasPension === false`.
**השלכה:** עובד ללא קרן פנסיה יראה עלות מעסיק עם פיצויים, בעוד שבפועל אין לו כיסוי פיצויים דרך קרן.
**תיקון מוצע:** `const sev = emp.hasPension ? gross * SEV : 0;`

### BUG-PEN-02 (גבוה)
**מיקום:** `src\App.jsx:59–60`
**תיאור:** חישוב `gross * PEN_E` ללא `Math.min(gross, PEN_CAP)`.
**השלכה:** עובד בשכר 40,000 ₪ יחושב עם 6% × 40,000 = 2,400 ₪ עובד, בעוד שלפי חוק ההפרשה מוגבלת ל־2 × שכר ממוצע במשק (≈24,730 ₪ ב־2025).
**תיקון מוצע:** הוספת `const PEN_CAP = 24730` ו־`const pensionable = Math.min(gross, PEN_CAP);`

### BUG-PEN-03 (קריטי)
**מיקום:** `src\App.jsx:14` (פונקציית `taxCalc`)
**תיאור:** חישוב המס לא מפחית את הטבת המס בגין הפרשות לפנסיה (סעיף 45א׳/47).
**השלכה:** עובד מפסיד זיכוי של עד 35% מהפרשותיו (≈ 6,300 ₪ בשנה למתפרש ממוצע).
**תיקון מוצע:** הוספת `const pensionTaxCredit = Math.min(penE, maxAllowed) * 0.35;` והפחתה מהמס.

### GAP-PEN-04 (קריטי)
**תיאור:** חסרה לחלוטין שכבת דיווח לגופים מוסדיים (אין מחולל XML "ממשק אחיד", אין שליחה לקרנות).
**השלכה:** אי אפשר להשתמש במערכת בייצור — דיווח חודשי לקרן הוא חובה חוקית.

### GAP-PEN-05 (גבוה)
**תיאור:** מודל הנתונים (`employees`) חסר שדות `fundType`, `fundProvider`, `policyNumber`, `joinDate`, `section14`.
**השלכה:** אי אפשר להבחין בין קרן פנסיה חדשה / ותיקה / ביטוח מנהלים / קופ"ג.

### GAP-PEN-06 (בינוני)
**תיאור:** אין הבחנה בין "שכר מבוטח" (Pensionable Salary) לבין שכר כולל. כל הברוטו מבוטח.
**השלכה:** רכיבים שאינם פנסיוניים (אש"ל, נסיעות לעיתים, ביגוד) נכללים שלא כדין בבסיס ההפרשה.
