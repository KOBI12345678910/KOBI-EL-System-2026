### 2.3. Bituach Leumi Rates 2026 (ביטוח לאומי + בריאות)

**חוק רלוונטי:** חוק הביטוח הלאומי [נוסח משולב] תשנ"ה-1995; חוק ביטוח בריאות ממלכתי תשנ"ד-1994.

**מה מופיע בקוד (שורה 10):**
```javascript
const BL_T=7122, BL_L=.035, BL_H=.12, HT_L=.031, HT_H=.05;
```

זאת אומרת:
- **סף ביטוח לאומי נמוך (BL_T):** 7,122 ₪
- **ביטוח לאומי מופחת (BL_L):** 3.5%
- **ביטוח לאומי מלא (BL_H):** 12%
- **ביטוח בריאות מופחת (HT_L):** 3.1%
- **ביטוח בריאות מלא (HT_H):** 5%

**בעיות חמורות שזוהו:**

1. **הסף שגוי:** הסף הנמוך הרשמי ל-2025 הוא 60% מהשכר הממוצע במשק (~7,522 ₪ ב-2025). הערך 7,122 **אינו תואם את הסף הרשמי של 2025 או 2026**. ייתכן שזה שארית מ-2023-2024.

2. **חסרה תקרה עליונה:** ב-2026 יש תקרה מקסימלית לדמי ביטוח לאומי (~49,030 ₪ - בצפי עדכון). הקוד:
```javascript
const blCalc=g=>Math.min(g,BL_T)*BL_L+Math.max(0,g-BL_T)*BL_H;
```
מחשב 12% על **כל** הסכום מעל הסף - **ללא תקרה עליונה**. זה **שגוי משפטית** ויגרום לגביית יתר עבור בעלי שכר גבוה.

3. **שיעורי עובד בלבד - חסרים שיעורי מעסיק:** לחוק יש שיעורי מעסיק נפרדים (3.55% מופחת, 7.6% מלא ב-2025). אין זכר לכך בקוד. בשורה 64:
```javascript
const erCost = gross + penR + sev + bl*1.05;
```
הקוד מכפיל את ניכוי העובד ב-1.05 כ-proxy לעלות מעסיק - **שגוי לחלוטין**.

4. **אין הפרדה בין ביטוח לאומי לביטוח בריאות של מעסיק:** לפי החוק, ביטוח בריאות משולם **רק על ידי העובד**, ולא על ידי המעסיק. הקוד לא מבדיל.

**ציון: 2/10** - שגיאות חישוב מהותיות, סף שגוי, חוסר תקרה, חוסר שיעורי מעסיק.

### 2.5. Severance Pay Law (חוק פיצויי פיטורים תשכ"ג-1963)

**חוק רלוונטי:** חוק פיצויי פיטורים תשכ"ג-1963.

**מה מופיע בקוד (שורה 11):**
```javascript
const SEV=.0833
```
ובשורה 61:
```javascript
const sev = gross * SEV;
```

**הערכה:**
- 0.0833 = 1/12, כלומר 8.33% - זה תואם ל"חודש פיצויים לשנת עבודה" (חישוב לפי סעיף 1 לחוק).
- **נכון מבחינת שיעור.**

**בעיות חמורות בכל זאת:**

1. **אין הפרדה בין הפרשות חודשיות לפיצויים במסגרת פנסיה (6% / 8.33%) לבין צבירת זכות מלאה לפי החוק.** סעיף 14 לחוק מתיר שחרור ממעסיק באמצעות הפקדה לקופת פיצויים - הקוד לא מבדיל.

2. **אין מעקב אחרי שנות ותק** - הפיצויים מחושבים רק על הברוטו של החודש הנוכחי, ללא התחשבות בבסיס שכר משרה קבועה או ותק.

3. **אין חישוב השלמה בעת פיטורים** - אם המעסיק הפריש פחות מ-8.33% לפנסיה, אין מנגנון להשלים בעת פיטורים.

4. **אין מעקב אחרי Section 14** - קובץ ה-employee לא מכיל שדה hasSec14 (האם חתום על סעיף 14).

5. **אין חישוב "פיצויי פרישה ברוטו"** מול "פיצויי פרישה מעל תקרה פטורה ממס" (סעיף 9א לפקודת מס הכנסה).

**ציון: 3/10** - שיעור נכון, אך אין תמיכה בדרישות המורכבות של החוק.

### 2.6. Vacation Days Law (חוק חופשה שנתית תשי"א-1951)

**חוק רלוונטי:** חוק חופשה שנתית תשי"א-1951.

**מה מופיע בקוד (שורה 50):**
```javascript
const vacPay = daysVacation * hourlyRate * 8.5;
```

**הערכה:**
- הקוד משלם "ימי חופשה" לפי שעות*תעריף שעתי. זה **ניסיון פשטני** אך לא תואם את החוק.

**בעיות חמורות:**

1. **אין ניהול צבירת זכאות חופשה.**
- לפי סעיף 3 לחוק: 12 ימי חופשה ב-4 שנים ראשונות, 14 בשנה ה-5, וכו' עד 23 יום בשנה ה-14.
- **אין לוח עדכני של צבירה בקוד.**
- **אין כל שדה vacationBalance ב-employee object.**

2. **אין הבדל בין ימי חופשה ברוטו (רגילים) לבין ימי חופשה בלתי מנוצלים (שצריך לפדות).**

3. **אין חישוב "דמי חופשה" לפי סעיף 10 לחוק** - חייב להיות לפי ממוצע השכר ב-3 חודשים האחרונים. הקוד מחשב רק לפי תעריף שעתי של החודש הנוכחי - **שגוי משפטית**.

4. **אין הפרדה חופשה שנתית / חופשה מיוחדת (לידה, אבלות, מילואים).**

5. **אין כפל חופשה בימי מנוחה שבועיים/חגים** (סעיף 7).

**ציון: 2/10** - חישוב פשטני, חסר מנגנון צבירה, חסר ממוצע 3 חודשים.

### 2.7. Sick Pay Law (חוק דמי מחלה תשל"ו-1976)

**חוק רלוונטי:** חוק דמי מחלה תשל"ו-1976.

**מה מופיע בקוד (שורה 49):**
```javascript
const sickPay = daysSick <= 1 ? 0 
              : daysSick <= 3 ? hourlyRate*8.5*(daysSick-1)*0.5 
              : hourlyRate*8.5*2*0.5 + hourlyRate*8.5*(daysSick-3);
```

**הערכה - ניתוח פרשני:**
- יום 1: 0% (לא משולם) - **תואם חוק** (סעיף 2(א): יום 1 לא משולם).
- ימים 2-3: 50% (שורה 49: `(daysSick-1)*0.5`) - **תואם חוק** (סעיף 2(ב): ימים 2-3 ב-50%).
- מיום 4 ואילך: 100% (שורה 49: `hourlyRate*8.5*(daysSick-3)`) - **תואם חוק** (סעיף 2(ג)).

**זה החלק היחיד בקוד שהוא יחסית תקין!**

**בעיות משניות אך חשובות:**

1. **אין ניהול צבירת ימי מחלה** - לפי סעיף 4 לחוק: 1.5 יום לחודש עד 90 יום. אין שדה sickBalance.

2. **אין אימות חופפת מחלה/חופשה** - הקוד יכול לשלם גם sick וגם vacation באותו יום (אם הוזנו שתי רשומות).

3. **אין הפניה לאישור רפואי** (נדרש מעל 2 ימים לפי תקנות).

4. **אין הפרדה מחלה אישית / מחלת ילד / מחלת בן משפחה** (חוק מחלת ילד, חוק טיפולי פוריות).

5. **אין שימוש בממוצע שכר שלושה חודשים** (סעיף 5 לחוק).

**ציון: 6/10** - החלק הכי קרוב לחוק, אבל עדיין פרימיטיבי.

### 2.8. Overtime Law (חוק שעות עבודה ומנוחה תשי"א-1951)

**חוק רלוונטי:** חוק שעות עבודה ומנוחה תשי"א-1951; תקנות שעות עבודה ומנוחה (זכויות בגין שעות נוספות), תש"ס-2000.

**מה מופיע בקוד (שורות 11-12, 38-44):**
```javascript
const OT_125=1.25, OT_150=1.50, OT_200=2.0;
// ...
records.filter(r=>r.status==="present"||r.status==="partial").forEach(r => {
    const hrs = r.hoursWorked || 0;
    if (hrs <= 8.5) { totalRegularHrs += hrs; }
    else if (hrs <= 10) { totalRegularHrs += 8.5; totalOT125 += hrs - 8.5; }
    else if (hrs <= 12) { totalRegularHrs += 8.5; totalOT125 += 1.5; totalOT150 += hrs - 10; }
    else { totalRegularHrs += 8.5; totalOT125 += 1.5; totalOT150 += 2; totalOT200 += hrs - 12; }
});
```

**הערכה - ניתוח מפורט:**

1. **יום עבודה = 8.5 שעות:** שגוי. לפי החוק (2018 ואילך), יום עבודה הוא **8.6 שעות** (43 שעות שבועיות / 5). הקוד מחשב 8.5. סטייה של 0.1 שעה ליום - זה ~2.5% פער.

2. **עיקרון 125% על 2 שעות ראשונות, 150% אחר כך:** **תואם חוק** (תקנות שעות עבודה ומנוחה, שעות נוספות).

3. **שיעור 200% - לא קיים בחוק הישראלי הבסיסי:**
- בישראל אין שיעור 200% לשעות נוספות. יש 150% בלבד מעבר לשעתיים נוספות.
- השיעור 200% קיים רק עבור **עבודה ביום המנוחה השבועי** (שבת), וגם אז בתוספת חוזרת (150% + 50% גמול מיוחד).
- הקוד **ממציא** קטגוריה 200% ללא בסיס חוקי.

4. **אין הפרדה שעות נוספות ביום חול / במנוחה השבועית / בחגים:**
- חוק קובע גמולים שונים לשעות נוספות בשבת (150% + תוספת 50% של שעות רגילות).
- הקוד לא מבדיל - כל השעות הולכות לאותה דלי.

5. **אין אימות שבוע עבודה של 42-43 שעות:**
- סעיף 2 לחוק: לא יותר מ-43 שעות בשבוע (לעובד 5 ימים) או 42 (ל-6 ימים).
- הקוד לא בודק.

6. **אין treatment ל"עובדים בתפקידים הדורשים אמון אישי":**
- סעיף 30(א)(5) לחוק: עובדי אמון אישי פטורים. אין שדה isTrustedPosition.

**ציון: 4/10** - עיקרון 125/150 נכון, אבל שורה של בעיות מהותיות.
