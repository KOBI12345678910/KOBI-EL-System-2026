## 2. ניתוח מפורט לפי דרישות הביקורת

### 2.1 שיעורי ביטוח לאומי לעובד שכיר (דרישה: 2026 — מדרגה נמוכה 0.4% / מדרגה עליונה 7%)

**מיקום הקוד:** `src/App.jsx` שורות 10, 15

```js
const CP_VAL=242, BL_T=7122, BL_L=.035, BL_H=.12, HT_L=.031, HT_H=.05;
// ...
const blCalc=g=>Math.min(g,BL_T)*BL_L+Math.max(0,g-BL_T)*BL_H;
```

**פענוח:**
- `BL_T` = סף המעבר (Threshold) בין מדרגה תחתונה לעליונה = **7,122 ₪**
- `BL_L` = שיעור מדרגה תחתונה (Low) = **3.5%**
- `BL_H` = שיעור מדרגה עליונה (High) = **12%**

**אי-התאמה מהותית לדרישה:**

| פרמטר | דרישה 2026 | קוד בפועל | פער |
|------|-----------|-----------|-----|
| שיעור עובד מדרגה תחתונה | **0.4%** | 3.5% | **+3.1 נקודות אחוז** (חיוב יתר פי ~8.75) |
| שיעור עובד מדרגה עליונה | **7%** | 12% | **+5 נקודות אחוז** (חיוב יתר פי ~1.71) |

**מסקנה קריטית:** הנוסחה `blCalc` מערבבת בין שיעור **עובד+מעסיק המשולב** (3.5% + 12%) לבין השיעור המיוחס לעובד בלבד. נראה כי המתכנת שם את השיעורים המלאים (עובד+מעסיק) בתור "ניכוי מהעובד" — זו שגיאה חמורה שגורמת לניכוי-יתר משכר נטו של העובדים בעשרות עד מאות ₪ בחודש.

**הערה נוספת:** גם אילו השיעורים ייצגו נכונה את החלק של העובד בלבד, הם לא תואמים את 2025 (שלפי חוק ישראלי בתוקף ינואר 2024: 0.4% בתחתונה ו-7% בעליונה — זהים למבוקש ל-2026, מה שמרמז שהדרישה מצפה לאותם שיעורים שכבר בתוקף).

### 2.2 שיעורי מעסיק לביטוח לאומי (דרישה: 2026 — 3.55% תחתונה / 7.6% עליונה)

**מיקום הקוד:** `src/App.jsx` שורה 64

```js
const erCost = gross + penR + sev + bl*1.05;
```

**פענוח:**
- עלות המעסיק מחושבת כ: ברוטו + פנסיה מעסיק + פיצויים + `bl*1.05` (כלומר 105% מסכום ביטוח הלאומי של העובד)
- אין חישוב נפרד של חלק המעסיק לביטוח לאומי, אין קבוע `BL_ER_L` או `BL_ER_H`, אין ביטוי במסד הנתונים של הפרדה עובד/מעסיק.

**אי-התאמה מהותית:**

| פרמטר | דרישה 2026 | קוד בפועל | פער |
|------|-----------|-----------|-----|
| שיעור מעסיק מדרגה תחתונה | **3.55%** | לא מחושב נפרד | **חסר לחלוטין** |
| שיעור מעסיק מדרגה עליונה | **7.6%** | לא מחושב נפרד | **חסר לחלוטין** |
| מכפיל גס של "עלות ב.ל. למעסיק" | — | `*1.05` | קירוב שגוי — המעסיק משלם *יותר* מהעובד, לא פחות |

**מסקנה:** השורה `bl*1.05` היא "Magic Number" שגוי מהיסוד. נטל המעסיק אינו 5% יותר מנטל העובד — לפי הדרישה (3.55%/7.6%) מול (0.4%/7%), המעסיק משלם **כ-9 פעמים יותר** במדרגה התחתונה ו-**כ-1.09 פעמים יותר** במדרגה העליונה. המבנה הזה לא מאפשר דיווח נכון של עלות מעסיק לצרכי דיווח חשבונאי או שחזור.

### 2.3 מס בריאות (דרישה: 2026 — 3.1% תחתונה / 5% עליונה)

**מיקום הקוד:** `src/App.jsx` שורות 10, 16

```js
const HT_L=.031, HT_H=.05;
const htCalc=g=>Math.min(g,BL_T)*HT_L+Math.max(0,g-BL_T)*HT_H;
```

**בדיקה:**

| פרמטר | דרישה 2026 | קוד בפועל | תקין? |
|------|-----------|-----------|-------|
| שיעור בריאות תחתונה | 3.1% | 3.1% (HT_L=.031) | OK |
| שיעור בריאות עליונה | 5.0% | 5.0% (HT_H=.05) | OK |

### 2.4 תקרת הכנסה מבוטחת מקסימלית (Maximum Insurable Income)

**מיקום הקוד:** `src/App.jsx` שורות 10, 15-16

```js
BL_T=7122
const blCalc=g=>Math.min(g,BL_T)*BL_L+Math.max(0,g-BL_T)*BL_H;
const htCalc=g=>Math.min(g,BL_T)*HT_L+Math.max(0,g-BL_T)*HT_H;
```

**ממצא קריטי — בלבול מושגי חמור:**
הקבוע `BL_T` (Threshold) משמש כאן בעת ובעונה אחת עבור:
1. סף המעבר בין מדרגות (שצריך להיות 60% מהשכר הממוצע, כ-7,522 ₪)
2. תקרת ההכנסה המבוטחת (שצריך להיות **47,465 ₪** ב-2025, עם עדכון צפוי ב-2026)

**אין קבוע נפרד לתקרה.** פירוש הדבר: עובד שמקבל 100,000 ₪ ברוטו יחויב לפי הנוסחה ב:
```
Math.min(100000, 7122) * 0.035 + Math.max(0, 100000-7122) * 0.12
= 7122 * 0.035 + 92878 * 0.12
= 249.27 + 11145.36 = 11,394.63 ₪
```

בעוד החוק האמיתי אומר: מעל התקרה (~47,465 ₪) **לא משלמים בכלל** ביטוח לאומי. עובד בכיר יחויב ב-*הרבה* יותר ממה שצריך (חיוב-יתר של אלפי ₪ בחודש). **זה פגם חוקתי רגולטורי חמור.**

אין הגנה, אין `Math.min(g, BL_CEILING)`, אין בדיקה.

### 2.5 הבדלים לעצמאים (Self-employed)

**מיקום הקוד:** `src/App.jsx` שורות 82-94

```js
function autoCalcSub(sub, jobs) {

  const vat = sub.hasVAT ? totalBilled * VAT : 0;
  const withhold = totalBilled * sub.taxRate;
  const toPay = totalBilled + vat - withhold;
  // ...

```

**ממצא:** הפונקציה `autoCalcSub` מתייחסת ל**קבלני משנה** (sub-contractors) בלבד בצורת "חשבונית נכנסת", כלומר:
- מחשבת מע"מ (VAT=17% — גם זה שגוי; המע"מ ב-2024/2025 הוא 18%)
- מחשבת ניכוי מס במקור (`sub.taxRate`)
- **לא מחשבת בכלל ביטוח לאומי לעצמאים**

**החיסרון:**
- אין חישוב שיעורי ביטוח לאומי של עצמאי (שונים מהותית מעובד שכיר: ~2.87% תחתונה / 12.83% עליונה לשנת 2024)
- אין תקרת הכנסה לעצמאי (שגם שונה במעט)
- אין דיווח לטופס 6101
- אין נוסחת "הכנסה חייבת לצרכי ביטוח לאומי" (ניכוי 52% מההכנסה הממוצעת וכו')

**מסקנה:** מבחינת ביטוח לאומי לעצמאים — המערכת **לא מכסה את המקרה בכלל**. עצמאים נחשבים רק כ"ספקים חיצוניים" והמערכת רק מנכה להם את המע"מ ואת ניכוי המקור בלי לטפל בחבות הביטוח הלאומי האישית שלהם.

### 2.8 מסלול ביקורת לחישוב (Audit Trail)

**מיקום הקוד:** `src/App.jsx` שורות 99, 140-143, 487-500

```js
const KEYS = { ..., log:"pr-log-v2" };
const logEvent=useCallback(async(action,detail)=>{
  const entry={id:uid(),time:new Date().toISOString(),action,detail};
  setEventLog(prev=>{const n=[entry,...prev].slice(0,200);save(KEYS.log,n);return n;});
},[]);
```

**מה קיים:**
- רישום אירועים ב-eventLog (פעולות כלליות: "הרצת שכר", "עובד חדש", "נוכחות"...)
- שמירה ב-localStorage תחת `pr-log-v2` דרך `window.storage.get/set`
- הגבלה ל-200 רשומות בלבד (עם חתך FIFO — שורה 142 `.slice(0,200)`)
- תצוגה בלשונית "לוג" עם כפתור "נקה" (שורה 490 — **מוחק את כל הלוג ללא אישור או הרשאה**)

**מה חסר לצרכי ביקורת רגולטורית:**
1. **פרטי החישוב עצמו אינם נשמרים** — רק "הרצת שכר חודש פלוני, סה"כ X" אבל לא איזה שיעור ביטוח לאומי שימש, איזו תקרה, איזה סף. אי-אפשר לשחזר את תוצאת החישוב אם השיעורים השתנו אחר כך.
2. **אין חתימה / hash** שמונעת מניפולציה לאחור.
3. **הלוג מוגבל ל-200 רשומות** — מידע עתיק נמחק אוטומטית (לא עומד בתקנות שמירת תיעוד 7 שנים של רשות המסים).
4. **אחסון ב-localStorage בדפדפן בלבד** — אם המשתמש מנקה cache או עובר מחשב, כל הלוג אובד.
5. **כפתור "נקה"** — מוחק את כל הלוג באופן מיידי ללא אישור, ללא שמירת גיבוי, ללא הרשאה.
6. **אין שמירה של ברוטו/ניכויים/נטו לכל עובד לכל חודש** — `runResult` נשמר לחלק מהחישובים אבל לא יוצא ל-persistent storage (לא ראיתי save עליו ב-KEYS.runs).
7. **אין חותמת זמן מוסמכת (TSA)** — רק `new Date().toISOString()` מקומי שניתן לזיוף.
8. **אין user ID** בלוג — פעולות אנונימיות. לא ברור מי ביצע מה.

## 4. ציטוטים רלוונטיים מהקוד

**שורות 6-16 (מנוע החישובים המלא):**
```js
const TAX = [
  {max:7010,r:.10},{max:10060,r:.14},{max:16150,r:.20},{max:22440,r:.31},
  {max:46690,r:.35},{max:60130,r:.47},{max:Infinity,r:.50}
];
const CP_VAL=242, BL_T=7122, BL_L=.035, BL_H=.12, HT_L=.031, HT_H=.05;
const PEN_E=.06, PEN_R=.065, SEV=.0833, VAT=.17, WORK_DAYS=22, WORK_HRS=186;
const OT_125=1.25, OT_150=1.50, OT_200=2.0;

const taxCalc=(g,cp=2.25)=>{let t=0,p=0;for(const b of TAX){t+=(Math.min(g,b.max)-p)*b.r;p=b.max;if(g<=b.max)break;}return Math.max(0,t-cp*CP_VAL);};
const blCalc=g=>Math.min(g,BL_T)*BL_L+Math.max(0,g-BL_T)*BL_H;
const htCalc=g=>Math.min(g,BL_T)*HT_L+Math.max(0,g-BL_T)*HT_H;
```

**שורה 56-64 (שימוש בתוך פונקציית חישוב העובד):**
```js
const tax = taxCalc(gross, emp.creditPoints);
const bl = blCalc(gross);
const ht = htCalc(gross);
const penE = emp.hasPension ? gross * PEN_E : 0;
const penR = emp.hasPension ? gross * PEN_R : 0;
const sev = gross * SEV;
const totalDeductions = tax + bl + ht + penE;
const net = gross - totalDeductions;
const erCost = gross + penR + sev + bl*1.05;  // Magic multiplier
```

## 5. המלצות לתיקון (עדיפויות)
