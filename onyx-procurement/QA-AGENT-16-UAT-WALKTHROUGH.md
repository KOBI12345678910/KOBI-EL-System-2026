# PART 1 — התקנה ראשונית (פעם אחת, 15 דקות)

זה נעשה פעם אחת. אם כבר עשית, דלג ל-Part 2.

## 1.1 · Supabase — המחסן
1. לך ל-**supabase.com** → Sign in.
2. New Project → שם: `onyx-procurement` → סיסמה חזקה → Region: West EU (Ireland).
3. חכה 1-2 דקות עד שהפרויקט מוכן.
4. ⚙️ Project Settings → API → **העתק לצד:**
   - `Project URL` (https://xxx.supabase.co)
   - `anon public` key (eyJ...)
5. SQL Editor → New Query → תדביק את התוכן של `supabase/migrations/001-supabase-schema.sql` → Run.
   - **צפוי לראות:** `Success. ONYX Database Schema created successfully!`
   - **אם לא ראית:** שלב 1.1 נכשל. תעצור ותפתח את השגיאה.
6. New Query שני → תדביק את `supabase/migrations/002-seed-data-extended.sql` → Run.
   - **צפוי לראות:** `Success`.
7. Table Editor → לחץ `suppliers`.
   - **צפוי לראות:** **13 ספקים** (לא 15 כמו שכתוב במדריך ישן).
8. Table Editor → `subcontractors`.
   - **צפוי לראות:** **8 קבלני משנה**.

**עצור וודא:** אם אין 13 ספקים ו-8 קבלנים — הסיד נכשל. תבדוק שורות שגיאה ב-SQL Editor.

## 1.2 · Replit — המנוע
1. replit.com → Create Repl → Node.js → שם `onyx-procurement`.
2. מחק את `index.js` שנוצר אוטומטית.
3. העלה (Upload file):
   - `server.js`
   - `package.json`
4. New File → שם `.env` → תכתוב בתוכו:
   ```
   SUPABASE_URL=https://YOUR_PROJECT.supabase.co
   SUPABASE_ANON_KEY=eyJ...שלך
   PORT=3100
   ```
   **חשוב:** הפורט חייב להיות **3100** (לא 3000). אם אתה כותב 3000 — הדשבורד לא ידבר עם השרת.
5. בטרמינל (Shell) למטה: `npm install` → חכה שיסיים.
6. `npm start`.
   - **צפוי לראות:**
     ```
     🚀 ONYX PROCUREMENT API SERVER
     Port: 3100
     Supabase: ✅ Connected
     ```
7. פתח ב-דפדפן את ה-URL של Replit + `/api/status`.
   - **צפוי לראות JSON:** `{ "engine": "ONYX Procurement System", "status": "operational", "supabase": "connected" }`.

**אם ראית "not configured" או Error:** תחזור ל-.env, תוודא שאין רווחים, שה-SUPABASE_URL נכון.

## 1.3 · הדשבורד — המסך שלך
כאן מחכה לך בעיה — הדשבורד בקובץ `web/onyx-dashboard.jsx` מקונפג ל-**http://localhost:3100**. זה אומר:
- אם אתה פותח את הדשבורד על אותו מחשב שרץ עליו השרת — עובד.
- אם אתה פותח מ-מחשב אחר (או הדשבורד על Replit שונה) — **לא עובד**.

**המסקנה ל-UAT הזה:** תריץ את השרת מקומית על המחשב שלך (לא Replit), או תעדכן את שורה 3 ב-`onyx-dashboard.jsx` ל-URL של ה-Replit שלך לפני שאתה טוען אותו בכרום.

**סימן:** נקודה אדומה בצד ימין + "לא מחובר" במקום "פעיל".

**פתרון זמני:** תערוך ידנית את שורה 3 ל-URL של ה-Replit שלך **לפני** שתרענן את הדשבורד. או — תריץ את השרת מקומית על המחשב שלך (`npm start` ב-cmd).
**הוכחה שזה הבאג:** טאב 2 (ספקים) ריק, טאב 1 מראה 0 בכל מקום, טוסט כלום לא קופץ.

## 3.B · PORT=3000 במקום 3100 (באג B-01 — תוקן ב-guide, אבל אם שיחקת עם .env)
**מתי:** אם שמת ב-.env משהו אחר מ-3100.
**סימן:** קונסול של כרום מראה "ERR_CONNECTION_REFUSED" ב-localhost:3100.
**פתרון:** תערוך את `.env` → `PORT=3100` → עצור את השרת (Ctrl+C) → `npm start` שוב.

## 3.C · "לא נמצאו ספקים לקטגוריות" ב-RFQ
**מתי:** אם ב-RFQ הוספת קטגוריה שאין לה אפילו מוצר אחד בסיד.
**סימן:** טוסט אדום "לא נמצאו ספקים לקטגוריות: XXX".
**פתרון:** תוריד את הקטגוריה הבעייתית, נסה שוב.
**הקטגוריות שבטוח יש עליהן ספקים (מהסיד):** ברזל, אלומיניום, נירוסטה, זכוכית, צבע.

## 3.D · "החלטה" מחזיר priceScore=0 (באג F-04)

**למה:** `server.js:460` מחשב `priceRange = maxPrice - minPrice || 1` — עם הצעה אחת זה 0→1, ואז priceScore יוצא 0.
**פתרון:** תמיד תזין לפחות 2-3 הצעות ב-RFQ.

## 3.E · הזמנה מסומנת "sent" גם כששליחה נכשלה (באג F-02)

## 3.H · אף API לא מוגן — כל אחד עם URL יכול לשלוח RFQ בשמך (באג B-03)

## 3.I · קריסה אם תריץ שוב את 002-seed (באג F-06)
