# Onyx Procurement — Bilingual Email System

**Agent:** 73
**Module path:** `src/emails/`
**Status:** additive, zero new npm dependencies, 24/24 unit tests passing.

This document describes the bilingual email template system, how to render
messages, and how to dispatch them through the pluggable sender.

Every template ships with:

- **Hebrew subject** and **English subject** (both use `{{var}}` substitution).
- **Hidden preheader** text (short preview shown by most inboxes).
- **HTML body** — RTL shell, inline CSS, mobile-friendly, branded header +
  footer, Hebrew primary content, English secondary section.

  Hebrew + English mailing address, unsubscribe link.
- **Disclaimer**: "automated message, do not reply" in both languages.

## Rendering a template

```js
const { renderTemplate } = require('./src/emails/email-templates');

const out = renderTemplate('wage_slip_issued', {
  employee_name: 'דנה כהן',
  period:        '2026-03',
  net_amount:    '₪12,345.67',
  gross_amount:  '₪16,800.00',
  pay_date:      '2026-04-10',
  slip_id:       'SLP-20260410-001',
});

// out = {
//   name:       'wage_slip_issued',
//   subject:    '...',          // Hebrew
//   subject_en: '...',          // English
//   preheader:  '...',
//   html:       '<!doctype html>...',  // full mail shell
//   text:       '...',                  // plain fallback
//   missing:    [],                     // any {{vars}} the caller forgot
//   category:   'payroll',
// }
```

### Variable substitution

`{{variable_name}}` inside any string field is substituted with the value in
the `vars` object.  `replaceVariables()` supports dotted paths
(e.g. `{{brand.name}}`).  Missing values are **not** silently replaced —
they stay as the literal token and are reported in `out.missing` so callers
can fail loudly in tests.

All HTML substitutions are **HTML-escaped** by default; plain-text
substitutions are not.  This prevents accidental HTML injection from
vendor or customer names without forcing callers to pre-escape data.

### Branding override

Pass `vars.brand` to override any default in `DEFAULT_BRAND`:

```js
renderTemplate('payment_confirmation', {
  ...data,
  brand: {
    name: 'Acme Corp',
    logo_url: 'https://acme.test/logo.svg',
    support_email: 'ops@acme.test',
    primary_color: '#0b5394',
  },
});
```

Unset fields fall back to the Onyx defaults.

## Sending mail

```js
const { createSender } = require('./src/emails/send-email');
const { renderTemplate } = require('./src/emails/email-templates');

const sender = createSender({
  transport: {
    type: 'smtp',
    host: 'smtp.example.com',
    port: 587,
    auth: { user: 'apikey', pass: process.env.SMTP_PASS },
    from: { name: 'Onyx Payroll', email: 'payroll@onyx.local' },
  },
  auditFile: '/var/log/onyx/email-audit.jsonl',
  maxRetries: 3,
  retryBaseDelayMs: 500,
  retryMaxDelayMs: 30_000,
});

const rendered = renderTemplate('wage_slip_issued', vars);

await sender.send({
  to: { name: vars.employee_name, email: 'employee@example.com' },
  subject: rendered.subject,
  html: rendered.html,
  text: rendered.text,
  attachments: [
    { filename: 'slip.pdf', content: pdfBuffer, contentType: 'application/pdf' },
  ],
}, { correlationId: 'slip-20260410-001' });
```

### Transports

| `transport.type` | Behaviour |
| ---------------- | --------- |
| `smtp`           | Real SMTP dialog via `net` / `tls`. Supports STARTTLS upgrade and `AUTH PLAIN`. |
| `sendgrid`       | Stub — builds MIME and returns a queued response. Real HTTP client must be wired separately. |
| `mailgun`        | Stub — same as SendGrid. |
| `ses`            | Stub — same; real implementation would call `aws-sdk` v3. |
| `noop`           | Composes MIME but never sends. Used in tests and local dev. |

All stub transports accept `dryRun: true` to avoid throwing when credentials
are missing — they still return a deterministic `accepted: true` response so
CI runs without network access.

### Retry + back-off

Each `sender.send(...)` call enqueues a job and drains the queue.
On transport errors the queue retries with exponential back-off
(`baseDelayMs * 2^(attempt-1)`, capped at `maxDelayMs`).  After
`maxRetries` the job is dead-lettered and an `email.dead_letter` audit
record is written before the promise rejects.

For fire-and-forget fan-out, use `sender.enqueue(message)` then call
`sender.drain()` from a worker.

### Audit log

Every send attempt writes a JSONL record to `auditFile` (or an in-memory
ring buffer when no file is configured).  Events:

| event                   | written when |
| ----------------------- | ------------ |
| `email.sent`            | transport accepted the message |
| `email.attempt_failed`  | individual attempt threw; will retry if attempts left |
| `email.dead_letter`     | exhausted all retries, job terminated |

Each record carries `timestamp`, `transport`, `to`, `subject`, `messageId`,
`attempts`, `queueId`, and any caller-provided `meta`.

## Compliance & privacy notes

- All emails include an **unsubscribe URL** in the footer (configurable
  through `brand.unsubscribe_url`).
- Transactional templates carry an **automated-message disclaimer** so
  recipients know not to reply directly.
- Hebrew and English text are both present, satisfying Israeli consumer
  protection expectations for bilingual communications.
- HTML is table-based with inline CSS to work in Gmail, Outlook, Apple
  Mail, and the Israeli webmail clients (Walla, 012).
- No PII is logged unencrypted — the audit log stores only recipient
  addresses, subjects, and message IDs, never body content.
