<a name="quick-start"></a>
## 1. התחלה מהירה (Quick Start)

### מה אני רואה
מסך ראשי עם שישה כפתורים גדולים: רכש, מע"מ, מס שנתי, בנק, שכר, הגדרות.
בראש המסך: לוגו "טכנו-קול עוזי", תאריך עברי, סטטוס חיבור לרשות המסים (ירוק/אדום).

### מה אני עושה
1. פתח את דפדפן Chrome או Edge.
2. גש לכתובת: `http://localhost:3000` (מקומי) או כתובת ה-Cloud שהוגדרה.
3. התחבר עם שם המשתמש והסיסמה שהוגדרו ב-`.env` (שדה `ADMIN_USER` / `ADMIN_PASS`).
4. לחץ על הכפתור של המודול הרצוי.

**טיפ:** אם זו פעם ראשונה — עבור קודם ל־**הגדרות → פרופיל עסק** והזן את פרטי הח.פ. שלך.

## 2. מודול רכש (Procurement Module) — onyx-procurement

מודול הרכש מאפשר יצירת בקשות מחיר (RFQ), השוואת הצעות, החלטה על ספק, הפקת הזמנת רכש (PO) ושליחתה בוואטסאפ.

### 2.1 יצירת בקשת מחיר (RFQ Creation)

#### מה אני רואה
טבלה של בקשות מחיר קיימות, כפתור ירוק "+ בקשת מחיר חדשה" בפינה השמאלית העליונה.

#### מה אני עושה
1. לחץ **"+ בקשת מחיר חדשה"**.
2. מלא את הטופס:
   - **שם הפריט** (למשל: "פלטת פלדה 10 מ"מ").
   - **כמות** (למשל: 50 יחידות).
   - **מפרט טכני / הערות**.
   - **תאריך יעד לקבלת הצעות**.
3. בחר ספקים מהרשימה (או הוסף ספק חדש בלחיצה על "+ ספק").
4. לחץ **"שלח RFQ"** — המערכת שולחת אימייל/וואטסאפ לכל הספקים הנבחרים.

### 2.2 השוואת הצעות (Quote Comparison)

#### מה אני רואה
טבלת השוואה אוטומטית עם עמודות: ספק, מחיר ליחידה, מחיר סה"כ, זמן אספקה, תנאי תשלום, ציון (score).

#### מה אני עושה
1. פתח את ה-RFQ מהרשימה.
2. עבור ללשונית **"הצעות מחיר"**.
3. המערכת מחשבת ציון אוטומטי (מחיר + זמן + אמינות היסטורית).
4. סקור את הטבלה ולחץ **"בחר ספק"** ליד הספק המועדף.

### 2.3 החלטה על ספק והפקת הזמנת רכש (Decision & PO Generation)

#### מה אני רואה
מסך אישור עם סיכום ההזמנה, סה"כ לפני/אחרי מע"מ, ותנאי התשלום.

#### מה אני עושה
1. ודא שהפרטים נכונים.
2. לחץ **"הפק הזמנת רכש (PO)"**.
3. המערכת מייצרת PDF עם מספר הזמנה ייחודי.
4. לחץ **"שלח בוואטסאפ"** — בחר את איש הקשר אצל הספק ושלח.

> **דרישת חוק:** הזמנת רכש נשמרת לארכיון למשך 7 שנים (חוק מע"מ סעיף 66).

<a name="vat-module"></a>
## 3. מודול מע"מ (VAT Module)

מודול זה מאפשר הגדרת פרופיל עוסק, הזנת חשבוניות (תשומות ועסקאות), סגירת תקופה, יצירת קובץ PCN836, והגשה לרשות המסים.

### 3.1 הגדרת פרופיל עוסק (Profile Setup)

#### מה אני רואה
טופס עם שדות: ח.פ./עוסק מורשה, שם עסק, תיק במס הכנסה, שיעור מע"מ (17%), סוג עוסק (רגיל / פטור / זעיר).

#### מה אני עושה
1. עבור **מע"מ → הגדרות → פרופיל עוסק**.
2. מלא:
   - ח.פ. / ת.ז. של העוסק.
   - שם עסק: "טכנו-קול עוזי".
   - סוג עוסק: **מורשה**.
   - שיעור מע"מ ברירת מחדל: **17%**.
3. לחץ **"שמור"**.

### 3.2 הוספת חשבוניות (Add Invoices)

#### מה אני רואה
שני טאבים: **"עסקאות" (מכירות)** ו**"תשומות" (קניות)**. בכל טאב — כפתור "+ חשבונית חדשה".

#### מה אני עושה
1. לחץ **"+ חשבונית חדשה"**.
2. מלא:
   - **מספר חשבונית**.
   - **תאריך**.
   - **ח.פ. לקוח/ספק**.
   - **סכום לפני מע"מ**.
   - **שיעור מע"מ** (ברירת מחדל: 17%).
   - **סכום מע"מ** (מחושב אוטומטית).
3. העלה סריקה / PDF (אופציונלי אך מומלץ).
4. לחץ **"שמור"**.

> **רפורמת חשבונית ישראל 2024:** חשבוניות מעל 25,000 ₪ דורשות מספר הקצאה (Allocation Number) — המערכת מבקשת אותו אוטומטית מרשות המסים.

### 3.3 סגירת תקופה והפקת PCN836

#### מה אני רואה
לחצן **"סגור תקופה"** עם רשימת כל החשבוניות של התקופה (חודשית / דו-חודשית).

#### מה אני עושה
1. עבור **מע"מ → סגירת תקופה**.
2. בחר תקופה (למשל: חודש 04/2026).
3. סקור את סיכום הנתונים: סה"כ עסקאות, סה"כ תשומות, מע"מ לתשלום / להחזר.
4. לחץ **"הפק PCN836"** — המערכת יוצרת קובץ טקסט בפורמט של רשות המסים.
5. לחץ **"הגש לרשות המסים"** — המערכת מתחברת ל-SHAAM API ומגישה אוטומטית.

<a name="annual-tax-module"></a>
## 4. מודול מס שנתי (Annual Tax Module) — onyx-ai

מודול זה מטפל בפרויקטים, לקוחות, חשבוניות, תקבולים, חישוב שנת מס, והפקת טפסים 1301 / 1320 / 6111.

### 4.1 ניהול פרויקטים (Projects)

#### מה אני רואה
רשימת פרויקטים עם סטטוס (פעיל / סגור), לקוח, תאריך התחלה, סה"כ הכנסות.

#### מה אני עושה
1. עבור **מס שנתי → פרויקטים**.
2. לחץ **"+ פרויקט חדש"**.
3. מלא: שם פרויקט, לקוח, תאריך התחלה, תקציב.
4. **מקרה שימוש:** "בניין רחוב X תל אביב — פרויקט ריתוך" / "פרויקט נדל"ן — בניין משרדים".

### 4.2 לקוחות (Customers)

#### מה אני רואה
טבלת לקוחות עם ח.פ., כתובת, יתרת חוב.

#### מה אני עושה
1. עבור **מס שנתי → לקוחות**.
2. לחץ **"+ לקוח חדש"**.
3. מלא: שם, ח.פ., כתובת, טלפון, אימייל, תנאי תשלום (שוטף+30 וכו').

### 4.3 חשבוניות ותקבולים (Invoices & Payments)

#### מה אני רואה
רשימת חשבוניות לפי לקוח, עם סטטוס תשלום (שולם / פתוח / באיחור).

#### מה אני עושה
1. **חשבונית:** לחץ "+ חשבונית" → בחר לקוח → הוסף פריטים → לחץ "הפק".
2. **תקבול:** לחץ "+ תקבול" → בחר חשבונית → הזן סכום ותאריך → לחץ "שמור".

### 4.4 חישוב שנת המס (Compute Fiscal Year)

#### מה אני רואה
מסך סיכום שנתי עם: הכנסות, הוצאות, רווח נקי, חישוב מס מקדמה.

#### מה אני עושה
1. עבור **מס שנתי → סגירת שנה**.
2. בחר שנת מס (למשל: 2025).
3. לחץ **"חשב שנת מס"** — המערכת מריצה את המנוע המלא.
4. סקור את התוצאה.

### 4.5 הפקת טפסים 1301 / 1320 / 6111

#### מה אני רואה
שלושה כפתורים: **1301** (דוח שנתי יחיד), **1320** (נספח), **6111** (דוח מאוחד).

#### מה אני עושה
1. לחץ על הטופס הרצוי.
2. המערכת מייצרת PDF מלא לפי פורמט רשות המסים.
3. הורד, חתום והגש (אונליין או ידנית).

<a name="bank-module"></a>
## 5. מודול בנק (Bank Module)

### 5.1 הוספת חשבון בנק (Add Account)

#### מה אני רואה
רשימת חשבונות עם יתרה נוכחית, סטטוס סנכרון אחרון.

#### מה אני עושה
1. עבור **בנק → חשבונות**.
2. לחץ **"+ חשבון חדש"**.
3. מלא: שם בנק (פועלים / לאומי / דיסקונט), מספר סניף, מספר חשבון, מטבע (₪).

### 5.2 יבוא דף חשבון (Import Statement)

#### מה אני רואה
איזור העלאה עם תמיכה בשני פורמטים: **CSV** או **MT940**.

#### מה אני עושה
1. עבור **בנק → יבוא**.
2. גרור את הקובץ (או לחץ "בחר קובץ").
3. המערכת מזהה אוטומטית את הפורמט.
4. בחר את החשבון ולחץ **"יבא"**.

### 5.3 התאמה אוטומטית (Auto-Reconcile)

#### מה אני רואה
טבלה של טרנזקציות מהבנק ליד טבלה של חשבוניות/תקבולים מהמערכת. התאמות שנמצאו מסומנות בירוק.

#### מה אני עושה
1. לחץ **"הרץ התאמה אוטומטית"**.
2. המערכת מנסה להתאים על בסיס סכום, תאריך, והערות.
3. התאמות ודאיות מסומנות אוטומטית.

### 5.4 אישור התאמות (Approve Matches)

#### מה אני רואה
רשימת התאמות ממתינות לאישור (סטטוס: "suggested"), וטופס להתאמה ידנית.

#### מה אני עושה
1. סקור כל התאמה מוצעת.
2. לחץ **"אשר"** (ירוק) או **"דחה"** (אדום).
3. התאמות לא נמצאו — לחץ "התאם ידנית" ובחר ידנית.

<a name="payroll-module"></a>
## 6. מודול שכר (Payroll Module) — payroll-autonomous

מודול זה מטפל בניהול מעסיק, עובדים, חישוב תלושי שכר, אישור והפקת PDF. תואם **חוק הגנת השכר תיקון 24**.

### 6.1 הוספת מעסיק (Add Employer)

#### מה אני רואה
טופס מעסיק עם: שם, ח.פ., ת.ד.מ. (תיק ניכויים), כתובת, בנק לתשלום.

#### מה אני עושה
1. עבור **שכר → מעסיקים**.
2. לחץ **"+ מעסיק חדש"**.
3. מלא את כל השדות (תיק ניכויים הוא **חובה**).
4. לחץ **"שמור"**.

### 6.2 הוספת עובד (Add Employee)

#### מה אני רואה
טופס עובד: פרטים אישיים, פרטי בנק, תלוש צפוי, קבצי 101.

#### מה אני עושה
1. עבור **שכר → עובדים**.
2. לחץ **"+ עובד חדש"**.
3. מלא:
   - **פרטים אישיים:** שם, ת.ז., כתובת, תאריך לידה.
   - **פרטי העסקה:** תאריך התחלה, סוג (חודשי / שעתי), שכר בסיס, תפקיד.
   - **פרטי בנק:** בנק, סניף, חשבון.
   - **נקודות זיכוי:** ברירת מחדל 2.25, עדכן לפי טופס 101.
   - **הפרשות:** פנסיה (6% עובד / 6.5% מעסיק / 8.33% פיצויים).
4. העלה טופס **101 חתום**.
5. לחץ **"שמור"**.

### 6.3 חישוב תלוש שכר (Compute Wage Slip)

#### מה אני רואה
מסך חישוב חודשי: רשימת עובדים, סה"כ ברוטו, סה"כ ניכויים, סה"כ נטו.

#### מה אני עושה
1. עבור **שכר → חישוב חודשי**.
2. בחר חודש (למשל: 03/2026).
3. עדכן שעות עבודה / בונוסים / החזרי הוצאות לפי הצורך.
4. לחץ **"חשב הכל"** — המערכת מריצה את מנוע החישוב המלא (ברוטו → ניכויים → נטו).
5. סקור את התוצאות בטבלה.

### 6.4 אישור תלוש (Approve Wage Slip)

#### מה אני רואה
טבלה עם עמודת סטטוס: "draft" (טיוטה) / "approved" (מאושר) / "paid" (שולם).

#### מה אני עושה
1. סקור כל תלוש.
2. לחץ **"אשר"** לכל תלוש שנכון.
3. או לחץ **"אשר הכל"** לאישור קבוצתי.

### 6.5 הפקת PDF (Issue PDF)

#### מה אני רואה
כפתור **"הפק תלושים ל-PDF"** ליד כל שורה או ברמת הקבוצה.

#### מה אני עושה
1. לחץ **"הפק תלוש PDF"**.
2. המערכת מייצרת תלוש חתום דיגיטלית לפי דרישות חוק הגנת השכר תיקון 24.
3. שלח לעובד באימייל / וואטסאפ או הדפס.

> **דרישת חוק חשובה:** התלוש חייב להכיל 17 שדות מינימום כולל: סוג תגמול, יום עבודה רגיל, שעות נוספות, הפקדות לקרן פנסיה, ימי חופשה / מחלה שנצברו.

<a name="security-api-keys"></a>
## 7. אבטחה ומפתחות API (Security & API Keys)

### מה אני רואה
עמוד הגדרות עם רשימת מפתחות API מוסתרים (מוצגים רק 4 תווים אחרונים).

### מה אני עושה
1. עבור **הגדרות → אבטחה**.
2. מפתחות נדרשים:
   - **SHAAM_API_KEY** — לחיבור לרשות המסים (PCN836 / הקצאת חשבוניות).
   - **WHATSAPP_API_TOKEN** — לשליחת PO בוואטסאפ.
   - **BANK_CONNECTOR_KEY** — לחיבור ל-Open Banking.
   - **JWT_SECRET** — אסימון הזדהות פנימי (סודי).
3. עדכן כל מפתח בלחיצה על "ערוך" (נפתח חלון מוצפן).
4. **אל תשתף את קובץ `.env` עם אף אחד.**
5. בצע גיבוי יומי של `.env` למיקום מוצפן (USB, ענן אישי).

### הרשאות ותפקידים
- **Admin** (קובי): גישה מלאה.
- **Viewer** (רואה חשבון): קריאה בלבד — ניתן להגדיר בהגדרות → משתמשים.

<a name="common-errors"></a>
## 8. שגיאות נפוצות (Common Errors)

| הודעה | משמעות | פתרון |
|---|---|---|
| `SHAAM_CONN_REFUSED` | אין חיבור לשרת רשות המסים | בדוק חיבור אינטרנט, ודא ש-`SHAAM_API_KEY` תקף |
| `INVALID_ALLOCATION_NUMBER` | מספר הקצאה שגוי | חשבונית מעל 25,000 ₪ — חזור לטופס והפק מספר חדש |
| `PCN836_FORMAT_ERR` | קובץ PCN836 לא תקין | סקור את החשבוניות — וודא שאין שדות ריקים |
| `EMPLOYEE_101_MISSING` | טופס 101 לא הועלה | עבור לעובד → העלה 101 חתום |
| `BANK_IMPORT_FAIL` | יבוא דף חשבון נכשל | בדוק פורמט (CSV / MT940), וודא שהקובץ לא פגום |
| `JWT_EXPIRED` | סשן פג תוקף | התחבר שוב |
| `DB_CONNECTION_LOST` | חיבור למסד נתונים נפל | הפעל מחדש את השרת: `npm run start` |
| `RFQ_NO_SUPPLIERS` | RFQ ללא ספקים נבחרים | חזור ובחר לפחות ספק אחד |

## 10. שאלות ותשובות (FAQ)

לרשימה מלאה של 30 שאלות נפוצות — ראה את הקובץ [`FAQ.md`](./FAQ.md).

**שאלות פופולריות:**

1. **ש:** האם המערכת עובדת גם במצב Offline?
   **ת:** כן, למעט הגשת PCN836 והקצאת חשבוניות — פעולות אלה דורשות חיבור לרשות המסים.

2. **ש:** מתי אני צריך לסגור תקופת מע"מ?
   **ת:** עד 15 לחודש העוקב (חודשי) או עד 15 לחודש הראשון אחרי תקופת הדו-חודש (דו-חודשי).

3. **ש:** האם המערכת מגבה את עצמה?
   **ת:** כן — גיבוי יומי אוטומטי ל-`./backups/`. רצוי להוסיף גיבוי ענן חיצוני.

4. **ש:** איך אני מוסיף משתמש חדש?
   **ת:** **הגדרות → משתמשים → "+ משתמש חדש"**. בחר הרשאה (Admin / Viewer).

5. **ש:** האם המערכת תומכת במספר מעסיקים (ח.פ. שונים)?
   **ת:** כן — כל מעסיק הוא ישות נפרדת במודול השכר עם חישוב עצמאי.
